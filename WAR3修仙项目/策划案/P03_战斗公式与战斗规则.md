# P03 战斗公式与战斗规则

> 版本：v1.0 ｜ 2026-06-17 ｜ 对应程序 S08/S19
> 定位：完整战斗伤害计算公式与流程。当前先整理完整内容，实际启用版本和启用范围另行确定。
> 属性来源：S属性配置表（attr_config）；公式来源：修仙角色属性-战斗公式流程.xlsx

---

## 一、伤害流程总览

每次攻击按以下顺序判定：

```
1. 命中判定（命中率 vs 闪避率）
2. 攻击取值（天运决定是否取上限）
3. 防御取值（天佑决定是否取上限）
4. 暴击判定（暴击率 vs 免暴击率）
5. 伤害计算（主公式 + 额外伤害）
```

---

## 二、命中判定

技能分三类，命中计算不同：

| 技能类型 | 基础命中率 | 最终命中率 |
|---------|-----------|-----------|
| 物理+法术伤害 | 95% | 基础命中 + 命中率 − 闪避率 |
| 控制 | 暂不考虑 | — |

- 命中率、闪避率均为万分比，计算时除以 10000
- 最终命中率上限 100%

---

## 三、攻击取值（天运）

根据天运值决定本次伤害计算使用的攻击数值。

### 3.1 天运触发几率

| 天运值 | 触发上限几率 |
|--------|------------|
| 0 | 0% |
| 1 | 11.11% |
| 2 | 12.50% |
| 3 | 14.29% |
| 4 | 16.67% |
| 5 | 20.00% |
| 6 | 25.00% |
| 7 | 33.33% |
| 8 | 50.00% |
| 9 | 100.00% |

### 3.2 取值流程

1. 根据天运值和上表几率，判定是否直接取攻击上限
2. 若未触发上限，在攻击下限～攻击上限之间随机取值（仍有可能随机到上限）
3. 伤害类型为物理 → 取攻击下限/攻击上限；伤害类型为法术 → 取法力下限/法力上限
4. 攻击下限增幅、攻击上限增幅（MODIFIER 类属性）由属性管理器预先计算，伤害公式直接读最终值

---

## 四、防御取值（天佑）

天佑决定防御方是否直接取防御上限，计算流程和数值表与天运完全一致。

1. 根据天佑值判定是否直接取防御上限
2. 若未触发上限，在防御下限～防御上限之间随机取值
3. 伤害类型为物理 → 取物防下限/物防上限；伤害类型为法术 → 取法防下限/法防上限
4. 物防下限增幅、物防上限增幅、法防下限增幅、法防上限增幅（MODIFIER 类）由属性管理器预先计算

---

## 五、暴击判定

- 暴击率 = 攻方暴击率 − 守方免暴击率（取值范围 0～100%）
- 暴击率、免暴击率均为万分比，计算时除以 10000
- 暴击后的伤害计算见下章公式

---

## 六、主伤害公式

### 6.1 未暴击伤害

**对精英 / BOSS：**

```
未暴击伤害 = (攻方攻击 − (守方防御 − 攻方无视防御) × (1 − 攻方无视防御增幅))
            × 法术威力
            × (1 + 法术增伤增幅 + (伤害提升增幅 − 守方伤害减免增幅)
               + (对BOSS增伤 − 守方对BOSS伤害减免))
            + 法术附带伤害
```

**对普通怪：**

```
未暴击伤害 = (攻方攻击 − (守方防御 − 攻方无视防御) × (1 − 攻方无视防御增幅))
            × 法术威力
            × (1 + 法术增伤增幅 + (伤害提升增幅 − 守方伤害减免增幅))
            + 法术附带伤害
```

### 6.2 暴击伤害

**对精英 / BOSS：**

```
暴击伤害 = 未暴击伤害 × (1 + 50% + 暴击伤害加成) + 暴击伤害增加
```

**对普通怪：**

```
暴击伤害 = 未暴击伤害 × (1 + 50% + 暴击伤害加成) + 暴击伤害增加
```

### 6.3 公式中各项说明

| 项 | 属性来源 | 说明 |
|----|---------|------|
| 攻方攻击 | 攻击下限/上限（物理）或法力下限/上限（法术） | 由天运取值流程确定 |
| 守方防御 | 物防下限/上限（物理）或法防下限/上限（法术） | 由天佑取值流程确定 |
| 攻方无视防御 | 无视防御（物理）或无视法防（法术） | 固定值穿透 |
| 攻方无视防御增幅 | 无视防御增幅（物理）或无视法防增幅（法术） | 百分比穿透，万分比 |
| 法术威力 | 法术威力 | 所有来源加总（装备/系统/当前技能等级），万分比 |
| 法术增伤增幅 | 法术增伤增幅 | 万分比 |
| 伤害提升增幅 | 伤害提升增幅 | 万分比 |
| 守方伤害减免增幅 | 伤害减免增幅 | 万分比 |
| 对BOSS增伤 | 对BOSS增伤 | 万分比 |
| 守方对BOSS伤害减免 | 对BOSS伤害减免 | 万分比 |
| 法术附带伤害 | 法术附带伤害 | 固定值 |
| 暴击伤害加成 | 暴击伤害加成 | 万分比 |
| 暴击伤害增加 | 暴击伤害增加 | 固定值 |

> 注意：法术威力、法术增伤增幅中的「法术」指广义的技能伤害系数，适用于物理和法术两种伤害类型，不是仅限法术攻击。

---

## 七、额外伤害

### 7.1 对精英 / BOSS

```
额外伤害 = 伤害提升 + (BOSS固定伤害 − 守方BOSS伤害减免) − 守方伤害减免
```

### 7.2 对普通怪

```
额外伤害 = 伤害提升 − 守方伤害减免
```

### 7.3 各项说明

| 项 | 属性来源 | 说明 |
|----|---------|------|
| 伤害提升 | 伤害提升 | 固定值 |
| BOSS固定伤害 | BOSS固定伤害 | 固定值 |
| 守方BOSS伤害减免 | BOSS伤害减免 | 固定值 |
| 守方伤害减免 | 伤害减免 | 固定值 |

---

## 八、最终伤害

```
未暴击最终伤害（精英/BOSS） = 未暴击伤害 + 额外伤害(精英/BOSS版)
未暴击最终伤害（普通怪）     = 未暴击伤害 + 额外伤害(普通怪版)
暴击最终伤害（精英/BOSS）    = 暴击伤害   + 额外伤害(精英/BOSS版)
暴击最终伤害（普通怪）        = 暴击伤害   + 额外伤害(普通怪版)
```

最终伤害最小值 ≥ 1。

---

## 九、约束规则

以下约束适用于公式中所有减法对，每一项单独判定，最小值为 0：

- `(守方防御 − 攻方无视防御)` ≥ 0
- `(1 − 攻方无视防御增幅)` ≥ 0
- `(伤害提升增幅 − 守方伤害减免增幅)` ≥ 0
- `(对BOSS增伤 − 守方对BOSS伤害减免)` ≥ 0
- `(BOSS固定伤害 − 守方BOSS伤害减免)` ≥ 0
- 额外伤害公式中的各减法对同样适用此规则

MODIFIER 类型属性（攻击下限增幅、攻击上限增幅、法力下限增幅、法力上限增幅、物防下限增幅、物防上限增幅、法防下限增幅、法防上限增幅、生命值增幅、灵力值增幅、攻法下限/上限及其增幅）由属性管理器预先计算，不直接出现在伤害公式中。伤害公式只读取属性管理器返回的最终 BASE 值。

万分比属性（命中率、闪避率、暴击率、免暴击率、暴击伤害加成、伤害提升增幅、伤害减免增幅、法术威力、法术增伤增幅、对BOSS增伤、对BOSS伤害减免）在公式中计算时除以 10000。

## 十、战斗节奏参数

### 10.1 普攻周期

```
前摇 0.30s → 命中帧(伤害计算+飘字) → 后摇 0.40s → 冷却 1.50s → 下一轮
完整周期 = 2.20s
这里的前摇，后摇，冷却时间最终读表，整理只是举例。
```

- 前摇结束触发伤害；后摇结束触发冷却开始
- 攻击范围（近战）：128 码（读配置，此地举例），范围外先移动到范围内再起手
- 冷却独立计算：切换目标不跳过冷却

### 10.2 攻击指令

选中怪物 → 右键/A 键 → 自动走到攻击范围 → 攻击循环。

## 十一、飘字规则

### 11.1 伤害飘字

| 属性 | 规则 |
|------|------|
| 内容 | 伤害数值整数（如 "8"） |
| 颜色 | 白色 RGB(255,255,255) |
| 时机 | 伤害计算完成、扣血生效时 |
| 位置 | 受击单位头顶 (0,+60) 像素 |
| 运动 | 向上匀速 ~40 像素，同时渐隐 |
| 时长 | 1.5s（前 0.5s 不透明，后 1.0s alpha 255→0） |
| 字号 | 12px |
| 叠加 | 多次伤害独立叠加，每次 +15 像素横向错开 |

### 11.2 V0 不做的飘字

暴击(黄字大字)/闪避/免疫/吸收/治疗(绿字)/技能名/境界压制——全部 V1-V4，V0 只有白色伤害数字。

## 十二、死亡与复活

这些内容大体上和以前差不多，以下只是补充性说明，不用严格按照下面的约束做。

### 12.1 死亡判定

每次扣血后立即检查 hp ≤ 0。同一帧内多次攻击按发起顺序依次扣血，死亡当帧后续攻击全部失效。

### 12.2 玩家死亡 → 复活

hp ≤ 0 → 播放死亡 → 3 秒后直接在当前场景复活点复活。hp=max_hp、mp=max_mp、移除死亡标记、恢复碰撞。

### 12.3 怪物死亡

| 项目 | 规则 |
|------|------|
| 死亡触发 | hp ≤ 0 当帧 |
| 死亡动画 | 约 1.0s |
| 掉落触发 | 死亡动画播放时（0.3s 内）触发掉落系统 |
| 尸体保留 | 动画结束后保留 5s，期间不可选中/不可攻击 |
| 尸体消失 | 5s 后约 0.5s 淡出 |
| 触发重刷 | 尸体消失后刷新点死亡计数+1，重刷倒计时开始 |

## 十三、给程序的关键提醒

只提醒不确定。

1. 伤害公式按本文第六章实现，不写公式字符串，不做公式解析器
2. 属性走 `get_attr(unit, attr_key)` 接口，不直接读配置表
3. MODIFIER 属性由 AttributeManager 预处理，伤害公式只取最终值
4. 扣血后立即判死，同帧不鞭尸
5. 前后摇用独立 Timer，不在动画回调里做战斗逻辑
6. 每个飘字独立生命周期 Timer
7. 怪物死亡→掉落→刷新由战斗系统发事件，掉落/刷新各自监听
8. 万分比属性统一除以 10000
