# V0 掉落规则

> 版本：v1.4 ｜ 2026-07-11 ｜ 对应程序 S12
> 归属：最后一击 / 保护 5s / 拾取 200 码。
> ⚠ **本文档中所有具体 ID 均为示例，实际以配置表（xlsx）为准。**

---

## 一、掉落触发

| 规则 | 定义 |
|------|------|
| 触发时机 | 怪物 hp≤0 当帧（死亡瞬间），死亡动画播放时 0.3s 内触发 |
| 触发次数 | 每怪一次 |
| 掉落来源 | 读 monster_config 的 `drop_methods`（array,int）→ 查 drop_config（P09） |
| 事件驱动 | S12 监听 S08 死亡事件，不在战斗代码里直接调掉落函数 |

---

## 二、独立概率掉落（32 槽）

`drop_config` 采用 **每槽独立按概率** 机制（`drop_type` = "掉落方式_按概率"）。

```
每个槽（共 32 个）独立判定：
1. 读取该槽的 掉落物概率（prob，单位 1/1e8，如 100000000 = 100%）
2. r = random(1, 100000000)
3. r ≤ prob → 该槽命中，产出 数量 份该物品
4. r > prob → 该槽不产出
5. 32 个槽全部独立过一遍，掉几个就是几个
```

## 三、归属判定

| 规则 | 定义 |
|------|------|
| 归属者 | 最后一击击杀怪物的玩家 |
| 保护期 | 5 秒，从掉落物生成起计 |
| 保护期内 | 仅归属者可拾，其他玩家提示"该物品归属他人" |
| 保护期后 | 任何人可拾 |
| 归属者断线 | 物品不受影响，继续存在至 60s；5s 后任何人可拾 |
| 无玩家击杀 | 无归属者，直接公开 |
| 归属时间 | 用服务器时间戳，防客户端改时间作弊 |

> V0 不做：队伍/公会共享、DPS 分配、轮流拾取、持有权转移。

## 四、掉落位置

```
单物品：坐标 = 怪物死亡坐标（偏移 0）
多物品：坐标 = 死亡坐标 ± 随机偏移（X/Y ∈ [-64,64] 码），半径约 64 码圆内散落
  → 每件独立随机，与已有地面物品距离<32 码则再偏移一次
障碍检测：目标点不可通行 → 向死亡点回缩 32 码，最多 3 次；仍不行 → 落死亡坐标
```

V0 灰盒为平地，障碍逻辑大概率不触发，但代码必须实现。

## 五、地面表现与消失

| 元素 | 规格 |
|------|------|
| 图标 | ground_model（空则复用 icon_id 缩小至 24×24），2D 面片始终面向镜头 |
| 悬浮文字 | 物品名，白色 12px，图标上方 (0,+20) |
| 选中高亮 | 鼠标悬停图标边缘白色轮廓 1px，文字加粗 |
| 存在时间 | 60 秒 |
| 闪烁 | 50-55s 开始闪烁（0.5s/次），55-60s 加速闪烁（0.25s/次）+ alpha 渐降至 0 |
| 超时 | 60s 未拾 → 销毁，不进背包、不补偿、不记日志 |
| 拾取后 | 立即消失 → 数据转入背包 → 服务器确认后广播 |

> V0 不做：品质颜色/光柱/3D 模型/弹射爆散/旋转/阴影。

## 六、防重复拾取（原子操作）

```
玩家点击地面物品 / 按 F 键（拾取范围 200 码）
  → 客户端 PICKUP_REQUEST(item_id, player_id)
  → 服务器原子操作：
      ① 物品仍在地面？
      ② flag_picked == false？
      ③ 归属通过？
      ④ 背包有空间？
  → 全过：flag_picked=true → 入包 → 广播消失 → PICKUP_SUCCESS
  → 任一失败：回失败原因 + 物品保持地面
```

服务器端是地面物品的唯一权威，客户端只有渲染镜像。
