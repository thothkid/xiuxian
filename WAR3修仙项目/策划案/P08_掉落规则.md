# V0 掉落规则（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P08 ｜ 对应程序 S12
> 融合来源：小马《V0_掉落规则》(主体,原子拾取/防作弊/障碍回缩/闪烁渐隐最完整) + 小龙《V0 掉落规则》(对比项)。
> **归属裁决（总分析）：最后一击 / 保护 5s / 拾取 200 码（取小马）。** 小龙的"首次伤害/10s/150码"是多人防抢机制，V0 灰盒单人为主用不上，已否决。

## 一、掉落触发

| 规则 | 定义 |
|------|------|
| 触发时机 | 怪物 hp≤0 当帧（死亡瞬间），死亡动画播放时 0.3s 内触发 |
| 触发次数 | 每怪一次 |
| 掉落来源 | 读 monster_config 的 drop_scheme_id → 查 drop_config（P09） |
| 事件驱动 | S12 监听 S08 死亡事件，不在战斗代码里直接调掉落函数（不硬耦合） |

## 二、权重池随机（32 槽）

```
1. 收集 weight>0 的槽，total_weight = Σweight
2. r = random(1, total_weight)（用 GetRandomInt，勿用 rand()%total 防模偏差）
3. 遍历累加 weight，r≤累计值 → 命中该槽
4. 数量 = random(count_min, count_max)
5. force_drop=true 的槽额外独立产出（V0 无此场景，全 false）
6. 本次掉落种类数 N = random(scheme.min_drops, scheme.max_drops)，重复抽 N 次
```

## 三、归属判定【裁决落地】

| 规则 | 定义 |
|------|------|
| 归属者 | **最后一击**击杀怪物的玩家（谁打死归谁） |
| 保护期 | **5 秒**（drop_claim_time），从掉落物生成起计 |
| 保护期内 | 仅归属者可拾，其他玩家拾取提示"该物品归属他人" |
| 保护期后 | 任何人可拾 |
| 归属者断线 | 物品不受影响，继续存在至 60s；5s 后任何人可拾 |
| 无玩家击杀（环境伤害） | 无归属者，直接公开 |
| 归属时间 | 用**服务器时间戳**，防客户端改时间作弊 |

> V0 不做：队伍/公会共享、DPS 分配、轮流拾取、持有权转移。

> **[V1 起]** 本节「最后一击 / 5s 保护」归属模型被 §九 三档策略（PUBLIC / PER_PLAYER / HOST）取代；V1 起保护默认「永久」。V0 单人下 PUBLIC 与本节 last-hit 跑出来等价，故 V0 保留本节不动。

## 四、掉落位置

```
单物品：坐标 = 怪物死亡坐标（偏移 0）
多物品：坐标 = 死亡坐标 ± 随机偏移（X/Y ∈ [-64,64] 码），半径约 64 码圆内散落
  → 每件独立随机，与已有地面物品距离<32 码则再偏移一次
障碍检测：目标点不可通行 → 向死亡点回缩 32 码，最多 3 次；仍不行 → 落死亡坐标（必可通行）
```

V0 灰盒为平地，障碍逻辑大概率不触发，但代码必须实现。

## 五、地面表现与消失

| 元素 | 规格 |
|------|------|
| 图标 | ground_model（空则复用 icon_id 缩小至 24×24），2D 面片始终面向镜头 |
| 悬浮文字 | 物品名，白色 12px，图标上方 (0,+20) |
| 选中高亮 | 鼠标悬停图标边缘白色轮廓 1px，文字加粗 |
| 存在时间 | 60 秒 |
| 闪烁 | 50-55s 开始闪烁（0.5s/次），55-60s 加速闪烁（0.25s/次）+ alpha 渐降至 0 |
| 超时 | 60s 未拾 → 销毁，不进背包、不补偿、不记日志；向归属者提示"掉落物已消失" |

> V0 不做：品质颜色/光柱/3D 模型/弹射爆散/旋转/阴影。

## 六、防重复拾取（原子操作）

```
玩家点击地面物品 / 按 F 键（拾取范围 200 码）
  → 客户端 PICKUP_REQUEST(item_id, player_id)
  → 服务器原子操作（同一锁内完成 4 步校验）：
      ① 物品仍在地面？
      ② flag_picked == false？
      ③ 归属通过？（保护期内=归属者 / 保护期后=任何人）
      ④ 背包有空间？（可堆叠 / 有空槽）
  → 全过：flag_picked=true（先标记防重复）→ 移出地面列表 → 入包 → 广播消失 → 回 PICKUP_SUCCESS
  → 任一失败：回失败原因 + 物品保持地面（拾取失败不消耗，倒计时不变）
```

服务器端是地面物品的唯一权威，客户端只有渲染镜像，所有拾取在服务器校验。

> **[灵石例外·V0 起]** 灵石（CURRENCY）落地后保留掉落表现（散落动画），随即自动入货币栏，不需点击、不走上述四步校验，不占格、不过滤、不受满包。详见 §9.3 与 P01 豁免。

## 九、V1 归属增量

> 版本：V1 ｜ 2026-06-26 ｜ 基于 V0 叠加。本章为 V1 起的归属与拾取机制，取代 §三 last-hit 模型。命名按 snake_case 规划，存量旧名（DropMethod/DurT 等）逐步迁移。

### 9.1 归属策略（配在掉落源/怪身上）

| 策略 | 谁能拾取 | 适用 |
|------|---------|------|
| `PUBLIC` | 所有人（无主） | 普通怪、环境击杀 |
| `PER_PLAYER` | 每个击杀者各自一份，互不可抢 | 普通 BOSS |
| `HOST` | 仅房主（生成时快照） | 稀有 BOSS、独有 BOSS |

- 最后一击概念在 PvE 整条删除：触发掉落、刷新计数、经验都不靠补刀。补刀只在 PvP 回归（道争录/斗法，V7）。
- HOST 的房主 = 掉落物生成那刻快照；房主中途换/退，归属不变。

### 9.2 拾取判定（三关流水线）

落地一件东西，按顺序过三关，任一关不过 = 拒、留地面：

```
归属 → 过滤器 → 满包 → 入包
```

- **归属**：判资格——这件你有没有资格拿。策划配死，玩家改不了。
- **过滤器**：判意愿——你要不要拿。玩家自己在设置里勾（如"灵石丹药收、狼牙破布不收"），每个号独立。
- **满包**：无空格、无可叠 → 拒，留地面。

> 先权限后意愿：没资格的（别人的 HOST 极品），到不了过滤器这一步。

### 9.3 宝宝（自动）vs 手点（手动）

| | 能拿范围 | 过滤器 | 满包 |
|------|---------|--------|------|
| 宝宝·自动 | 归属允许你拿的 | 必须放行才抓 | 满则跳过，留地面 |
| 手点·手动 | 归属允许你拿的 | 无视（手动=我就要） | 满则拒，提示 |

- 过滤器只挡宝宝：勾了不收破布，宝宝不去捡；手点那块破布照样拿得走。
- 按归属分流：`PUBLIC` → 所有人宝宝都来抓；`PER_PLAYER` → 各人宝宝抓各人那份；`HOST` → 只房主宝宝抓得到，别人够不着。
- **灵石永远自动收**，不占格、不过滤、不受满包（V0 起生效，见 §六例外）。

### 9.4 两个时间字段（放的层不一样）

| 字段（规划名） | 管什么 | 绑谁 | V1 值 |
|------|--------|------|------|
| `protect_time` | 归属保护：先归我、多久后转公开 | 怪/掉落源 | 全填「永久」 |
| `drop_lifetime` | 落地后多久消失 | 物品/装备 | 每件自己定 |

- `protect_time` 绑怪：同一把铁剑，普通怪掉的和 BOSS 掉的归属规则不同——归属是「这次掉落」的属性、跟着怪走，不是物品固有属性。
- 现定稿：永久归属、无尾巴，没人捡到点直接消失，中途不会变别人能捡。
- 字段留位、现全填「永久」。判定逻辑读字段、别写死；「过了 protect_time → 转公开」代码现在就写好、只是永久时永远走不到。以后想要尾巴：某类怪改成「5 秒」，零改代码。
- `drop_lifetime` 绑物品：极品该躺久、垃圾该早消失，放 item_config/装备表，全局只留一个默认兜底。

### 9.5 击杀推送

BOSS 击杀向全体玩家推送（谁杀的、掉了什么、归谁）——稀有/独有 BOSS 尤其，全场看见才有眼馋感与稀有事件仪式感。普通怪不推（防刷屏）。**[V3+]**

### 9.6 配置落地（三层，谁放哪）

| 层 | 表 | 字段（规划名） | 值 |
|------|------|------|------|
| ① 物品/装备 | item_config / 装备表 | `drop_lifetime` | 每件自己定（极品配「很长/不消失」） |
| ② 怪物 | monster_config | `drop_owner_policy` | PUBLIC / PER_PLAYER / HOST，每怪必填 |
| ② 怪物 | monster_config | `protect_time` | 永久 / 空（继承全局），选填 |
| ③ 全局 | drop_global_config | `protect_time_default` | 永久 |
| ③ 全局 | drop_global_config | `drop_lifetime_default` | 60 |

读取顺序（给程序）：落地 → 归属策略 + protect_time 看这只怪（怪没填取全局）→ 消失时长看这件物品（没填取全局兜底）→ 拾取走 §六原子四步（服务器权威、按件存 owner）。

> 通用原则：每种怪可能不一样的（策略）跟怪走；绝大多数一样、个别例外的（数值）全局默认 + 表里可覆盖。

### 9.7 BOSS 三档与三层世界（V3+ 归档）

| 档 | 刷新 | 归属 | 住哪层世界 |
|------|------|------|------|
| 普通 BOSS | 正常 | PER_PLAYER（各拿各的） | 房间实例世界 |
| 稀有 BOSS | 正常但间隔很久 | HOST（归房主） | 房间实例世界 |
| 独有 BOSS | 击杀后永不刷新 | HOST（归房主） | 个人存档世界 |

- 三层世界 / 独有 BOSS 编码沿用开发底座「原则二·三层世界边界」与「J3·独有BOSS」，本章不另定义，仅引用。
- HOST 对非房主：**显示地面掉落但不可拾，提示「归属房主」**（"别人只能看着"）。
- V0/V1 无 BOSS 载体，本节整段标 **[V3+]**，随版本推进再落地。

### 9.9 经验（修为）

砍怪不给修为（修为靠打坐/闭关，在个人存档世界）。挂机蹭经验整条作废，无需任何门槛。以后若做，模型用场景/队伍共享（不靠击杀者），怕蹭就限"同队"。这是 V2 的事（P01 Nope：修为 V2）。

### 9.10 V3+ 待办（已定细则，随归档段落地）

- PER_PLAYER 多人：掉 **N 份独立物品**（各人可见各人的 + 按 owner_id 过滤拾取），非影子副本。
- HOST 对非房主：显示 + 文案「归属房主」（见 §9.7）。
- HOST-BOSS 极品 `drop_lifetime` 默认配长/不消失（否则房主满包/漏捡，极品照样 60s 蒸发）。
