# V0 物品表（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P10 ｜ 对应程序 S13/S14
> 融合来源：小马《V0_物品表》(10 种全字段/与老 W物品表对照/CSV·JSON 主体) + 小龙《V0 物品表》(并入字符串枚举风格 + WC3 地面模型码)。
> 定位：V0 共 10 种修仙物品，覆盖货币/消耗品/材料/武器/防具五类。无品质/无词条/无强化。表名 `item_config`（G3）。

---

## 一、设计原则

- 每种物品至少参与一条闭环链路（打怪→掉落→拾取→背包→使用/穿戴→效果验证）。
- 五类各至少 1 种，确保程序类型枚举跑通。
- 装备数值小而明确（铁剑 atk+3 / 布衣 def+2），保证 `max(atk-def,1)` 可验证。
- V0 全部白色无品质、无词条、无强化、无即食掉落。

## 三、枚举定义

| 字段 | 取值 |
|------|------|
| `type` | CURRENCY / CONSUMABLE / MATERIAL / WEAPON / ARMOR |
| `func_type` | NONE / RESTORE_HP / RESTORE_MP / RESTORE_BOTH / EQUIP_ATK / EQUIP_DEF |
| `equip_slot` | NONE / WEAPON / ARMOR |

> 全部用字符串枚举（G2）：配置里 "WEAPON" 比 "4" 不易错，程序读 enum 字符串无额外成本。

## 四、物品功能逻辑

### 消耗品使用
```
右键消耗品 → 按 func_type 恢复：
  RESTORE_HP   → hp += func_value（上限 max_hp）
  RESTORE_MP   → mp += func_value（上限 max_mp）
  RESTORE_BOTH → hp/mp 各 += func_value
→ 数量 -1（归零移除）→ 绿色飘字 "+{func_value}"
满状态使用：提示"气血充盈/灵气充沛，无需使用"，不消耗
```

### 装备穿戴（详见 P12）
```
右键装备 → 按 equip_slot 入位 → 玩家 atk/def += func_value → 从背包移除
装备位已占 → 先卸旧（属性恢复）回背包，再穿新；背包满 → 替换失败，旧装备保留
```

### 货币
灵石进货币栏（HUD/背包底部计数），不占背包格。

## 五、地面表现

- ground_model 优先用上表的 WC3 模型码；为空时复用 icon_id 缩小至 24×24 面片（始终面向镜头）。
- 悬浮物品名白色 12px，存在 60s（50s 闪烁，55s 渐隐）。详见 P08。

## 六、配置表字段定义（item_config）

| 字段名 | 类型 | 必填 | 默认 | 说明 |
|--------|------|:--:|------|------|
| `item_id` | int | ✅ | — | 唯一 ID，编码 1xxx=货币/2xxx=消耗品/3xxx=材料/4xxx=装备 |
| `item_name` | string | ✅ | — | 中文名 |
| `type` | enum | ✅ | — | CURRENCY/CONSUMABLE/MATERIAL/WEAPON/ARMOR |
| `stack_limit` | int | ✅ | 1 | 货币999/消耗品·材料99/内丹10/装备1 |
| `func_type` | enum | ✅ | NONE | 见枚举定义 |
| `func_value` | int | ❌ | 0 | 恢复量 或 装备属性加成 |
| `equip_slot` | enum | ❌ | NONE | WEAPON/ARMOR（仅装备类填） |
| `icon_id` | string | ✅ | — | 背包/UI 图标资源 ID（64×64） |
| `ground_model` | string | ❌ | 空(复用icon) | 地面模型码，空则复用 icon 缩小 |
| `sell_price` | int | ❌ | 0 | 售价（V0 无商店，预留） |
| `quality` | int | ❌ | 0 | 品质（V0 固定 0=白，V3 启用） |
| `level_require` | int | ❌ | 0 | 等级需求（V0 固定 0） |
| `description` | string | ❌ | — | Tooltip 描述文本 |
| `drop_lifetime` | float | ❌ | 空=继承全局兜底(60) | 物品级地面存活时长（秒）。落地后多久消失，每件自定；极品配长/不消失。V1 增量，见 P08 §9.4/§9.6 |

> `drop_lifetime` 为 V1 归属增量字段。V0 全部物品留空 → 走全局 `drop_lifetime_default`=60。规划名 snake_case；P17(v1.0) 骨架中对应列旧名为 `DurT`，改名随迁移逐步进行。

## 八、融合说明

| 维度 | 取自 | 理由 |
|------|------|------|
| 10 种物品集（含灵泉水/妖兽内丹） | 小马 | 灵泉水验"双恢复"分支、妖兽内丹(堆叠10)验"低上限满包"，验收价值高 |
| item_id 分段编码 | 小马 | 与 monster_id 同构、可扩展 |
| **type/func_type/equip_slot 字符串枚举** | 小龙 | 可读性强，配置不易错 |
| **ground_model 用 WC3 模型码** | 小龙 | 比"复用 icon"更直接可接，保留"空则复用 icon"为兜底 |
| 装备 +3/+2、回灵丹 MP+50、灵石堆叠 999 | 小马 | 与 P12/P03 自洽；MP+50 与聚气丹 HP+50 对称 |
| sell_price/quality/level_require/老表对照/CSV·JSON | 小马 | 字段更全 |

> 否决：小龙 8 种(缺灵泉水/内丹)、item_id 顺序 1-8、铁剑+5/布衣+3、灵石堆叠 9999、回灵丹 MP+30。
