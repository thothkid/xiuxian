# V0 背包规则（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P11 ｜ 对应程序 S14
> 融合来源：小龙《V0 背包规则》(分节骨架清晰,主体) + 小马《V0 背包规则》(并入满包替换边界细则)。
> 定位：V0 背包的格子/拾取/堆叠/满包/使用/存档规则。

## 一、背包规格

| 规则 | 定义 |
|------|------|
| 格数 | 6 格（默认显示），初始背包格数由 P14 init_char_config.bag_size 决定（V0=12）。可通过配置在 6-12 间调整。 |
| 快捷键 | B 键开关背包面板 |
| 格子尺寸 | 48×48px |
| 显示内容 | 物品图标 + 名称 + 堆叠数量（右下角角标） |
| 货币显示 | 灵石数量独立显示（不占格，在背包面板底部/HUD） |
| 装备栏 | 内嵌武器位 + 衣服位（另有 C 键独立角色面板） |

## 二、拾取规则

| 规则 | 定义 |
|------|------|
| 拾取方式 | 点击地面掉落物（V0）/ 走近自动拾取（V1） |
| 灵石例外 | 灵石（CURRENCY）V0 起落地后保留掉落表现，自动入货币栏，不需点击；不占格、不过滤、不受满包（见 P08 §六 / §9.3、P01 豁免） |
| 堆叠物品 | 已有同 item_id 且未达 stack_limit → 优先填满已有堆 |
| 新增物品 | 无同 item_id 或已达上限 → 占用空格 |
| 满包判断 | 无空格且无可叠加同类 → 拒绝拾取，提示"储物袋已满（背包已满）"，物品留地面 |
| 过滤器 | 玩家意愿勾选（收/不收），V1 起生效，只挡宝宝自动拾取，不挡手点；详见 P08 §9.2/§9.3 |

### 入包算法
```
1. 遍历已有格子 → 同 item_id 且 count<stack_limit → 合并
2. 遍历空槽 → 填入剩余数量
3. 仍有余量 → "背包已满" → 留在地面（详见 P08 拾取）
```

## 三、堆叠规则

| 类型 | 可堆叠 | 上限 |
|------|:--:|------|
| CURRENCY（灵石） | 进货币栏，不占格 | 999 |
| CONSUMABLE（丹药） | ✅ | 99 |
| MATERIAL（材料） | ✅ | 99（妖兽内丹 10） |
| WEAPON/ARMOR（装备） | ❌ | 1，每件占一格 |

堆叠数 >1 时图标右下角显示数字（如 "×3"）。

## 四、右键菜单（按类型）

| 物品类型 | 菜单选项 |
|---------|---------|
| CONSUMABLE | 使用 |
| WEAPON/ARMOR-未穿戴 | 穿戴 |
| WEAPON/ARMOR-已穿戴（装备位） | 卸下 |
| MATERIAL | 无操作（V0 仅查看） |
| CURRENCY | 不进背包，无菜单 |

## 五、满包处理（并入小马边界细则）

| 场景 | 处理 |
|------|------|
| 拾取时无空位无叠加 | 拒绝拾取，提示"背包已满"，物品留地面（60s 内不消失） |
| 满包时怪物死亡 | 掉落物正常生成于地面，不受背包状态影响 |
| **穿戴替换且背包满** | 旧装备无处可回 → 替换失败，旧装备保留装备位、新装备保留背包，**不做任何更改、不销毁** |
| **卸下且背包满** | 卸下失败，提示"背包已满，无法卸下装备"，装备保留装备位 |

> V0 绝不销毁物品/装备：任何因满包导致的操作失败都保持原状。

---

## 六、边界条件

| 场景 | 处理 |
|------|------|
| 使用最后一个物品 | 数量归零 → 格子清空 |
| 穿戴后原背包格 | 装备移出背包 → 该格释放 |
| 死亡时背包 | 不受影响，V0 无死亡掉落 |

## 七、存档字段（与 P16 对齐）

| 字段 | 类型 | 说明 |
|------|------|------|
| `backpack` | JSON | 背包格数组：`[{slot_index, item_id, count}]` |
| `weapon_item_id` | int | 武器位物品 ID（0=空） |
| `armor_item_id` | int | 衣服位物品 ID（0=空） |
| `spirit_stone` | int | 灵石数量（货币，不占格） |

> 字段名与 P12 装备、P16 存档统一（G3）。小龙原用 inventory_slots/equipment_weapon/currency_ling_shi，归一为 backpack/weapon_item_id/spirit_stone。
