# 法宝行为分类与隐藏分类体系

> 版本：v1.2 ｜ 2026-06-17
> 定位：法宝系统的完整设计骨架。融合三套分类体系，逐次审核后补齐七组行为缺口。

---

## 一、设计哲学

### 1.1 wonder 与 progression 两端

老修仙和现代修仙的装备体系裂成了两端：

**老派（八仙过海、封神、炼妖壶）——wonder（惊奇/博物/斗宝）**
- 定性奇效：葫芦「叫名字就收进去」、照妖镜「一照现原形」——能力是规则性的，没有「+多少攻击」
- 斗法是斗宝：谁的宝克谁的宝谁赢，修为次要，法宝是战斗主体
- 每件有名有来历有脾气，博物式的琳琅满目

**现代（凡人流之后）——progression（进度/分级/练级）**
- 法宝分层分级体系化，随境界淘汰
- 核心变成修为/境界攀爬，法宝降格为配套战力零件
- 可炼制、可交易、可量产——祛魅

**关键发现：老味道没全死，是「裂成两端、中间层塌了」**
- 极少数奇物升格为「金手指」：掌天瓶、囚笼——独一无二、贯穿全程。对应项目的**道器**。
- 绝大多数法宝降格为「体系装备」：分级、数值、量产。对应项目的**外物装备层**。
- 消失的是中间层——八仙过海那种「人人手持一件奇宝、斗的是宝的奇效」的层。这个塌掉的中间层，正是法宝系统要补回来的。

### 1.2 三层挂载，两套逻辑

| 层 | 数量 | 逻辑 | 绑定 | 生命周期 |
|----|------|------|------|---------|
| 外物装备 | 6 | progression（刷宝+替换+迭代） | 由来源和品质决定 | 按大境界淘汰 |
| 法宝 | N | wonder（奇效+身份） | 认主绑定 | 跨境界成长 |
| 道器 | 1 | wonder（生涯级金手指） | 不可换 | 单世生涯级，轮回重置 |

三层不共享竞争池，是三条平行的追求线。

- **外物层**走 progression——日常刷宝节奏，可持续供给的「求取→获得→变强」。是节奏主引擎。
- **法宝层**走 wonder——有奇效、可孕育器灵（非标配）、每个原型有独特规则；同原型实例因材料、祭炼、器纹和认主关系产生个体差异。是题材招牌和身份锚点。别做成高级装备。
- **道器层**走 wonder——掌天瓶级别的生涯金手指。装备位显示但点击打开独立系统面板。单世内不可替换；轮回后重置并重新随机。

> 法宝与道器的行为边界：两者可以同样催熟、探宝、炼妖，区别不在「能做什么」，而在——法宝需要配置、祭炼和使用，有条件、有代价；道器是生涯级系统规则，长期改变玩家的一条成长路径。

### 1.3 物件原型驱动

法宝不是换了个名字的饰品位。它们在被设计之前，已经在文化记忆里携带了完整的功能语义：

| 物件 | 天然联想 |
|------|---------|
| 葫芦 | 收纳、炼丹、孕剑、喊名字收人 |
| 伞 | 防御、飞行、遮天机 |
| 壶 | 炼化、收妖、自成洞天 |
| 印 | 镇压、封印、砸人 |
| 扇 | 呼风、点火、煽动 |
| 镜 | 照妖、反弹、洞彻 |
| 钟 | 震慑、召请、计时 |
| 塔 | 镇压、收纳、保护 |
| 鼎 | 炼丹、炼器、镇压气运 |
| 舟 | 渡界、飞行、携带 |
| 旗 | 布阵、号令、光环 |
| 绳 | 捆缚、控制、捕获 |

八仙过海一人一件，件件不同——不是因为身体部位不同，是因为每件东西本身就是一种玩法。

## 二、三套分类体系（互补共存）

以下三套分类体系是从不同维度对法宝功能的切分，不是互斥的「哪个对」，而是互补的「各从什么角度看」。

### 体系 A：按游戏行为域分类（47 子类，六大域）

最细粒度。每件法宝插入一个或多个具体游戏行为。适合策划逐件设计时做功能定位。

**第一域 · 战斗**
攻伐 | 护持 | 控制 | 辅助疗愈 | 身法遁行 | 召御 | 阵域 | 破法反制 | 变化幻化 | 协同共斗

**第二域 · 修行**
聚灵修炼 | 神识悟法 | 淬体养命 | 破境渡劫 | 功法神通增幅 | 压制破压制 | 灵根资质

**第三域 · 百艺生产**
炼丹 | 炼器 | 制符 | 布阵 | 合成转化 | 灵植 | 御兽 | 养宝御器

**第四域 · 探索获取**
采集勘探 | 探知鉴别 | 寻踪机缘 | 隐匿偷渡 | 收纳储运 | 空间传送 | 天时环境 | 封禁解锁 | 猎获掉落

**第五域 · 洞府经营与社会**
洞府经营 | 守府警戒 | 交易鉴宝 | 宗门权柄 | 派遣役使 | 记录传承 | 契约交涉 | 营造修复

**第六域 · 异道与高阶**
净化解毒 | 魔化风险 | 魂魄器灵 | 因果气运 | 寿元轮回

> 设计时注意：当前分类的命名维度并不统一——攻伐是目的，阵域是释放方式，炼丹是系统域。把它们当同级标签可能导致「这法宝到底归哪个」的犹豫。后续应改为**行为标签库**（动词 + 场景 + 形态）以降低歧义。目前暂不改动结构，待下一轮细化时统一处理。

### 体系 C：按传统器物功能分类（8+1 类）

从传统修仙文学和民间传说中直接继承的宝物分类方式。源自「地理博物志怪」传统。

| 分类 | 典型器物 | 天然功能联想 |
|------|---------|-------------|
| 收摄·炼化 | 葫芦、壶、瓶、鼎、斗、花篮 | 吸收/收纳/炼化/反吐 |
| 显照·探查 | 镜、灯、珠 | 照妖/反隐/鉴宝/看破 |
| 镇压·封禁 | 印、钟、塔、盘 | 封印/压制/驱散/镇守 |
| 困缚·控制 | 绳、索、图、卷 | 捆缚/拖拽/囚禁 |
| 护持·遮蔽 | 伞、旗、幡、莲台、荷花 | 防御/护盾/遮天机/光环 |
| 风雷·变化 | 扇、芭蕉扇、风袋、雷锤 | 呼风/点火/唤雷/元素 |
| 音律·召请 | 笛、渔鼓、玉板、拍板、铃 | 鸣响/buff/召请/驱兽 |
| 驭行·渡界 | 舟、云、梭、剑 | 飞行/瞬移/渡界/传送 |
| 攻伐（器兵） | 宝剑、飞剑、剪、针、锤、环 | 直接攻击/穿透/破甲 |

> 注意：宝剑、飞剑等器形按功能判断——普通可换代者进入「武器位」；有独特名称、来历和规则神通者作为法宝。

### 三套体系的关系

```
体系 C（8+1 类，器物直觉）
    ↓ 每类对应
体系 B（13 类，功能粗分）
    ↓ 每类展开为
体系 A（47 子类，行为精确定位）
```

- **体系 C** 用于命名和题材包装
- **体系 B** 用于 UI 和玩家认知
- **体系 A** 用于策划生产和程序实现

## 三、隐藏分类总则

**「万物皆可炼化为法宝」不能靠给每件物品手工创造一套完全独立机制实现，内容量会失控。**

应采用隐藏的多轴组合：

```
法宝 ＝ 器物外形 ＋ 主功能类 ＋ 副功能类(0-2个) ＋ 释放形态 ＋ 五行/材料特性 ＋ 一条标志性规则
```

每件法宝可以拥有：
- 1 个主分类（体系 A 子类）；
- 0–2 个副分类（体系 A 子类）；
- 1 个体系 B 归属（用于 UI 展示）；
- 1 个体系 C 器形大类（用于命名和题材）；
- 1 种主要战斗释放形态；
- 若干五行、材料、灵气特征；
- 1 条让玩家记住它的独特规则。

分类不必直接展示给玩家，只用于策划生产、技能复用和内容管理。

> **生成规则占位**（待细化）：法宝来源分天生和炼化两条路径。原物炼化为法宝时，能力由原物用途、物理形态、材料五行、经历因缘和炼化方式共同生成——伞偏护持、秤偏鉴别、磨盘偏镇压转化；镜生照射、钟生震荡、绳生捆缚；灵木偏生发、玄铁偏锋锐；久经香火的钟与战场遗留的旗产生不同规则；精血祭炼偏认主、魔核祭炼偏高收益反噬。具体映射表后续补。

## 四、体系 A 详细展开：六大域 47 子类

### 第一域：战斗法宝

#### 1. 攻伐类
**核心动词**：斩、刺、射、轰、灼、冻、震、蚀、爆、诛。
**游戏表现**：直接伤害 / 持续伤害 / 范围伤害 / 对单体爆发 / 连锁攻击 / 斩杀 / 穿透 / 破甲、破盾 / 五行伤害与生克 / 对妖、魔、魂等特定目标增伤
**常见器形**：剑、飞剑、剪、针、锤、印、钟、塔、珠、扇、旗。

#### 2. 护持类
**核心动词**：挡、护、避、吸、卸、转、替、返。
**游戏表现**：护盾 / 减伤 / 伤害吸收 / 将伤害转化为法力或法宝充能 / 抵挡一次致命攻击 / 分摊伤害 / 反射弹道 / 元素抗性 / 护体领域 / 短暂无敌 / 替劫、复生
**常见器形**：伞、盾、袍、珠、莲台、塔、镜、钟、图。

#### 3. 控制类
**核心动词**：困、缚、镇、压、定、封、摄、迷、逐。
**游戏表现**：定身 / 减速 / 眩晕 / 沉默 / 缴械 / 拉拽、击退 / 囚禁 / 降低攻击欲望 / 强制聚怪 / 打断施法 / 禁止位移 / 压制 BOSS 阶段能力
**常见器形**：绳、索、钟、塔、印、图、旗、镜、壶、盘。

#### 4. 辅助与疗愈类
**核心动词**：生、愈、补、净、振、赐、续。
**游戏表现**：恢复生命 / 恢复灵力 / 清除负面状态 / 提升攻击、防御、速度 / 增强队友 / 延长增益 / 减少技能冷却 / 战斗中生成恢复区域 / 将溢出治疗转化为护盾 / 临时提升神识、真元或抗性
**常见器形**：瓶、葫芦、灯、珠、莲台、笛、琴、伞、花篮。

#### 5. 身法与遁行类
**核心动词**：遁、移、闪、渡、飞、缩、穿、返。
**游戏表现**：冲刺 / 瞬移 / 闪烁 / 穿越单位或地形 / 御风、御剑飞行 / 脱离战斗 / 回城 / 跨区移动 / 留下残影或替身 / 位移后隐身 / 携带队友移动
**常见器形**：舟、梭、剑、伞、云、图、旗、靴、门、盘。

#### 6. 召御类
**核心动词**：召、役、放、化、育、附、驭。
**游戏表现**：召唤灵兽 / 召唤器灵 / 召唤傀儡 / 放出被收摄的妖物 / 生成临时分身 / 召唤元素或气灵 / 强化已有灵兽 / 将击杀目标短暂转化为战斗助手 / 让器物自身环绕攻击
**常见器形**：壶、葫芦、袋、幡、铃、鼓、鼎、图、偶、印。

#### 7. 阵域类
**核心动词**：布、展、聚、隔、禁、陷、迷、镇。
**游戏表现**：在地面生成持续区域 / 聚灵领域 / 防御领域 / 困阵、迷阵 / 陷阱 / 禁止传送 / 改变区域五行 / 强化友方或削弱敌人 / 法宝展开后成为临时阵眼 / 随角色移动的便携阵法
**常见器形**：阵盘、旗、幡、图、卷、印、塔、伞、灯。

#### 8. 破法与反制类
**核心动词**：破、解、截、夺、返、散、禁。
**游戏表现**：驱散敌方增益 / 净化己方减益 / 打断法术 / 吸收敌方技能 / 反射攻击 / 禁用敌方法宝 / 摧毁护盾 / 解除阵法和禁制 / 窃取短时增益 / 针对特定五行或法术进行克制
**常见器形**：镜、剪、印、钟、尺、针、剑、幡、灯。

#### 9. 变化幻化类（新增）
**核心动词**：变、化、仿、分、虚、实。
**游戏表现**：变形为动物或物体 / 复制敌方技能或形态 / 制造幻象迷惑敌人 / 分出可控分身 / 改变自身或敌人大小 / 虚实转换（暂时不可选中或穿越障碍）/ 将敌人变成其他形态（如变羊）
**常见器形**：扇、镜、图、珠、幡、偶、斗篷。
> 古典法宝中极重要的一类，现有隐匿类仅含伪装，变化维度在此补齐。

#### 10. 协同共斗类（新增）
**核心动词**：合、链、承、渡、叠。
**游戏表现**：法宝能力在队友之间连锁 / 多人合力祭出同一法宝 / 队伍共同充能 / 替队友承劫或分摊雷劫 / 两件法宝组合形成合击 / 队伍共享领域效果（收益不简单叠加）
**常见器形**：印、钟、旗、图、剑、灯。
> 项目有组队、共渡雷劫、共同秘境，此类覆盖多人协同场景。

### 第二域：修行法宝

#### 11. 聚灵修炼类
**核心动词**：聚、纳、藏、炼、运、温养。
**项目表现**：提高打坐修炼速度 / 提升洞府灵气上限 / 储存灵气，在低灵气时段释放 / 减少修炼中断损失 / 提高离线闭关收益 / 调节灵气五行比例 / 暂存溢出修为 / 加快法宝祭炼与认主 / 提升真元恢复
**世界观依据**：物质法则 §五——修士经脉和丹田是为天界格式化后的灵气设计的。（更正：原写§六有误）
**常见器形**：珠、瓶、葫芦、鼎、莲台、蒲团、灯、盘、图。

#### 12. 神识与悟法类
**核心动词**：观、照、悟、演、记、辨、推。
**项目表现**：提升隐藏探测范围 / 强化小地图感知 / 看破隐身、幻象和迷阵 / 提升法宝操控数量或效率 / 提高炼丹、画符的神识精度 / 加快功法参悟 / 记录敌方技能并提供提示 / 推演配方或阵法 / 提高心魔抵抗 / 降低丹毒源头产生量
**世界观依据**：物质法则 §十——神识精度决定炼丹丹毒源头；最表层 §六——神识=小地图+隐藏探测+法宝操控数。
**常见器形**：镜、书、卷、简、灯、珠、棋盘、尺、眼形玉器。

#### 13. 淬体养命类
**核心动词**：淬、固、养、生、换、续。
**项目表现**：提升生命与防御 / 加快伤势恢复 / 强化经脉承载 / 增加丹田容量 / 提高丹毒和魔气耐受 / 提高毒、火、寒、雷抗性 / 减少重伤与虚弱时间 / 辅助洗髓 / 延长寿元 / 提升雷劫承受能力
**常见器形**：鼎、炉、钟、珠、瓶、骨器、玉床、莲台。

#### 14. 破境渡劫类
**核心动词**：破、引、渡、镇、护、替、承。

功能分类解决「它做什么」，表现模板解决「它怎样打出来」。

### 攻击释放模板
直线投射 | 扇形散射 | 持续射线 | 连锁弹射 | 环绕飞行 | 近身旋斩 | 地面爆发 | 持续领域 | 延时陷阱 | 标记后引爆 | 吸收后反吐 | 召唤物攻击 | 锁定追踪 | 蓄力重击 | 条件斩杀

### 防御释放模板
瞬时格挡 | 持续护盾 | 环绕护体 | 展开结界 | 吸收转化 | 反射弹道 | 伤害分摊 | 替身承伤 | 濒死触发 | 受击积蓄后反击

### 控制释放模板
单体锁定 | 直线贯穿 | 圆形困阵 | 拉向法宝 | 击退扩散 | 周期震荡 | 标记延迟控制 | 地形封锁 | 神识迷惑 | 吞入法宝形成临时放逐

同一功能可以用不同模板表现。例如同为控制：
- 绳索表现为追踪捆缚
- 钟表现为周期震慑
- 塔表现为范围镇压
- 镜表现为照射定形
- 葫芦表现为吸入放逐

## 六、法宝生成公式

```
器物原型（体系 C）
+ 体系 B 主归属
+ 体系 A 主功能分类
+ 体系 A 副功能分类（可选 0-2 个）
+ 战斗释放模板
+ 五行/阴阳/魔气标签
+ 材料来源
+ 触发方式
+ 标志性规则
+ 代价或限制
```

### 示例

**吞火葫芦**
- 体系 C：收摄·炼化 → 体系 B：收摄 → 体系 A：控制（主）＋ 炼丹（副）
- 战斗表现：吸收前方火焰与投射物，蓄满后反吐火流
- 生产表现：炼丹时稳定火候，降低过火概率
- 标签：火、木、收摄
- 限制：吸收寒水类攻击会降低充能

**照骨镜**
- 体系 C：显照·探查 → 体系 B：显照 → 体系 A：探知鉴别（主）＋ 攻伐（副）
- 战斗表现：持续照射，显露隐身并逐层削弱护甲
- 世界表现：识别妖魔伪装、装备污染和药材年份
- 标签：金、光、神识
- 限制：镜面被攻击后暂时失效

**青藤万生伞**
- 体系 C：护持·遮蔽 → 体系 B：护持 → 体系 A：护持（主）＋ 灵植（副）
- 战斗表现：展开治疗与减伤领域，受击后藤蔓缠绕敌人
- 生产表现：洞府中提高灵植生长与采集完整度
- 标签：木、水、生发
- 限制：火属性环境中效果降低

**镇魂钟**
- 体系 C：镇压·封禁 → 体系 B：镇压 → 体系 A：控制（主）＋ 魂魄器灵（副）
- 战斗表现：周期震荡，普通敌人恐惧，精英与 BOSS 积累镇魂值后眩晕
- 修行表现：提高心魔抵抗，辅助兽魂保存
- 标签：金、音、魂
- 限制：连续敲击消耗大量神识

## 七、技术实现建议

不要把 47 个子类做成 47 套互不通用的程序系统。

程序和配置层应进一步抽象为若干通用效果模块：

伤害 | 治疗 | 护盾 | Buff / Debuff | 位移 | 召唤 | 区域 | 资源修改 | 概率修改 | 时间修改 | 产量修改 | 品质修改 | 损耗修改 | 探测与显隐 | 储存与释放 | 转化 | 条件触发

策划上的隐藏分类负责保证行为覆盖和题材差异；底层通用模块负责控制开发成本。

最关键的结构是：**器形不决定分类，分类也不限制器形。任何物体都可以炼成法宝，但炼成之后必须明确：它介入什么行为、以什么方式介入、玩家为什么会记住它。**

## 八、术语约定

| 术语 | 精确定义 |
|------|---------|
| 灵力 / 灵力值 | 修士体内可调用的灵气总量（=蓝量/资源池） |
| 法力 | 法术攻击力（=法术输出属性，决定法术伤害） |
| 修为 | 长期积累的修炼进度（=经验条），不可消耗 |
| 攻击 | 物理攻击力 |
| 天运 | 攻击时额外取上限值的几率（=幸运） |
| 天佑 | 防御时额外取上限值的几率 |
| 会心 | 暴击 |
| 法术 | 技能 |
| 冷却 | CD |

## 九、设计指导清单

1. 这件东西属于哪一层？（外物装备 / 法宝 / 道器）
2. 体系 C：它用的是什么物件原型？文化语义和游戏机制是否一致？
3. 体系 B：它的功能粗分类是什么？
4. 体系 A：它的主功能子类？副功能子类？
5. 它的战斗释放模板？非战斗表现？
6. 它有没有让玩家记住的标志性规则？
7. 它有什么代价或限制？
8. 它的世界观依据来自物质法则的哪一节？
