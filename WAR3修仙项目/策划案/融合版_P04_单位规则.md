## 一、单位总则

**单位（Unit）= 游戏中所有可交互实体的统一抽象。** V0 只有两种：

| 类型 | 枚举 | 说明 |
|------|------|------|
| 玩家 | `UNIT_TYPE_PLAYER` | 真人操控的修仙者。数据来自服务器存档→属性系统 |
| 怪物 | `UNIT_TYPE_MONSTER` | AI 驱动的敌对生物。数据来自 monster_config（P05） |

玩家和怪物**共用同一套基类**：标识/阵营/属性/生命状态/可攻击判定/模型绑定/属性接口完全相同，只是属性数据来源不同。后续版本扩展 `UNIT_TYPE_NPC=3` / `UNIT_TYPE_PET=4`。

## 二、单位字段

| 字段 | 中文 | 类型 | 玩家初值 | 怪物初值 | 说明 |
|------|------|------|---------|---------|------|
| `unit_id` | 单位ID | int | 服务器分配 | 运行时自增 | 运行时唯一标识，≠ 配置主键。建议玩家 1-999、怪物 1000+ |
| `unit_name` | 名称 | string | 角色名 | monster_name | 飘字/HUD/目标框显示 |
| `unit_type` | 类型 | enum | PLAYER | MONSTER | 决定阵营和属性来源 |
| `faction` | 阵营 | enum | FACTION_PLAYER | FACTION_MONSTER | 同阵营不可攻击 |
| `hp` / `max_hp` | 生命/上限 | int | 100 / 100 | monster.hp | 归零=死亡。V0 max_hp=初始 hp |
| `mp` / `max_mp` | 灵力/上限 | int | 50 / 50 | 0 / 0 | 怪物 mp=0（V0 无施法怪） |
| `atk` | 攻击 | int | 10 | monster.atk | 单值 |
| `def` | 防御 | int | 5 | monster.def | 单值 |
| `atk_range` | 攻击范围 | int | 128 | monster.atk_range | 不填用 combat_config.attack_range_melee(128) |
| `atk_speed` | 攻击间隔 | float | 1.5 | monster.atk_speed | 玩家=combat_config.attack_speed(1.5)；怪物读 P05 monster.atk_speed |
| `move_speed` | 移动速度 | int | WC3默认 | monster.move_speed | 码/秒 |
| `aggro_range` | 索敌范围 | int | — | monster.aggro_range | 仅怪物，P06 用 |
| `chase_range` | 追击范围 | int | — | monster.chase_range | 仅怪物，P06 用 |
| `model_id` | 模型 | string | HeroPaladin(占位) | monster.model_id | WC3 单位代码 |
| `model_scale` | 缩放 | float | 1.0 | monster.model_scale | — |
| `pos_x`/`pos_y`/`facing` | 坐标/朝向 | float | 出生点 | 刷新点 | 运行时位置 |
| `is_alive` | 存活 | bool | true | true | = (hp>0) |

> 配置主键（monster_id）≠ 运行时 unit_id：monster_id 是怪物表主键；unit_id 由服务器创建时自增分配。怪物的 atk_range/atk_speed/aggro_range/chase_range 全部读 P05 monster_config，本表不另维护数值（避免与 P05 冲突）。

### V0 预留字段（定义但不赋值，避免后期改基类）
`vision_range`(视野，V3) · `level`(等级，固定1，V2) · `realm`/`realm_sub`(境界，固定"练气一层"/"初期"，V2)。

## 三、阵营与可攻击判定

阵营敌对矩阵：

| 攻击方↓ / 守方→ | FACTION_PLAYER | FACTION_MONSTER |
|---|:--:|:--:|
| FACTION_PLAYER | ❌ | ✅ |
| FACTION_MONSTER | ✅ | ❌ |

- `faction` 由 `unit_type` 推导（PLAYER→FACTION_PLAYER，MONSTER→FACTION_MONSTER），不在配置表独立设置；配置表 faction 字段仅作显式声明/一致性校验。
- V0 无 PvP、无中立阵营（V5+ NPC 引入 FACTION_NEUTRAL）。玩家间、怪物间均不可攻击。

```cpp
bool IsAttackable(Unit* attacker, Unit* target) {
    if (attacker->hp <= 0) return false;       // 攻击者存活
    if (target->hp   <= 0) return false;       // 目标存活
    if (attacker->faction == target->faction) return false;  // 敌对阵营
    return true;
}
```

## 四、生命状态

```cpp
enum UnitState { UNIT_STATE_ALIVE=1, UNIT_STATE_DEAD=2 };
```

| 状态 | 条件 | 行为 |
|------|------|------|
| ALIVE | hp > 0 | 可攻击/被攻击/移动/被选中 |
| DEAD | hp ≤ 0 | 停移动/攻击，不可选中/锁定，取消碰撞，血条清空 |

- **死亡单位不立即从池中移除**：怪物死亡后保留在池中至尸体消失，期间 `IsAlive()=false` 阻止交互、防空指针。
- 死亡时序统一走 **P03 §3 + combat_config**：怪物死亡动画 **1.0s** → 尸体保留 **5s** → 淡出 0.5s → 重刷倒计时（P07）；玩家死亡 → 3s 后弹"回城复活" → 复活回出生点、hp/mp 回满、清怪物锁定。**死亡无惩罚**（不掉落/不掉修为/不掉灵石）。

> 收敛说明：小龙原稿写"怪物死亡动画 1.5s / 尸体 3s"，与已锁定的 P03（1.0s / 5s）冲突，本版按 P03 收敛，不取小龙值，保证 P03/P04/P06 时序一致。

## 六、单位生命周期（并入小龙）

```
玩家：进房 → 载入存档(或创角) → 单位创建+绑定属性
        → 游戏中(移动/战斗/拾取/存档) → 退出 → 自动存档 → 单位销毁

怪物：玩家靠近刷怪区 → 怪物创建+绑定属性(读 P05)
        → AI 循环(待机/索敌/追击/攻击/脱战/归位)
        → 死亡 → 触发掉落(一次) → 尸体消失 → 单位销毁
        → 重刷计时(P07) → 新怪创建
```

## 七、单位接口（供 P03/P06/HUD 调用）

| 接口 | 说明 | 调用方 |
|------|------|--------|
| `Unit_Create(config)` | 按配置创建实例，初始化全字段 | S06/S09 |
| `Unit_Destroy(unit_id)` | 销毁实例（尸体消失时） | S09 |
| `GetAttr(unit_id, key)` | 读属性 | P03/HUD |

| `IsAttackable(a, t)` | 可攻击判定 | P03 |
| `SetState(unit_id, state)` | 切 ALIVE/DEAD | P03(死亡触发) |
| `PlayAnimation(unit_id, anim)` | 播放动画 | P03(前后摇) |
| `GetModelId(unit_id)` | 取模型 | S06 渲染 |

## 九、程序实现要点 + 红线

1. 单位池 `map<int, Unit>`，unit_id 为 key，O(1) 查找。
2. 死亡单位保留至尸体消失（怪物 5s）再移除，期间 IsAlive=false。
3. 玩家 GetAttr 桥接 S07 属性系统；怪物直接读实例字段；接口签名统一，内部按 unit_type 分发。
4. max_hp/max_mp：V0=初始值；后续版本由属性公式计算。
5. 怪物 atk_range/atk_speed/aggro_range/chase_range 从 P05 读，不在 P04 另设数值。
- **红线**：严禁旧传奇字段名——atk 不是 DC/atk_min/atk_max；def 不是 AC/物防/魔防；mp 不是魔法值二元结构。
