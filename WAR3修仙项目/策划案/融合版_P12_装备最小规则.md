# V0 装备最小规则（融合版）

> ⚠ **本文档中所有具体 ID 均为示例，实际以配置表（xlsx）为准。**


> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P12 ｜ 对应程序 S15
> 融合来源：小马《V0_装备最小规则》(穿戴/替换/卸下全流程+满包边界+存档恢复+属性对照主体) + 小龙《V0 装备最小规则》(对比项)。
> 定位：V0 只做穿戴验证——2 装备位、2 种装备、加算属性、实时生效、存档一致。不做品质/套装/强化/词条。装备数值 **铁剑 atk+3 / 布衣 def+2**（G2 锁定）。

---

## 一、装备位定义

| 装备位 | slot_key | 可穿戴 | V0 测试装备 | 属性影响 |
|--------|----------|--------|------------|---------|
| 武器 | WEAPON | type=WEAPON 的物品 | 铁剑(4001) | atk +3 |
| 衣服 | ARMOR | type=ARMOR 的物品 | 布衣(4002) | def +2 |

- V0 仅 2 位。V3 扩展至 8 槽（武器/头盔/衣服/护腕/戒指/鞋子/腰带/项链）。
- 穿戴校验：物品 equip_slot 与装备位 slot_key 匹配，不匹配则拒绝并提示"该物品无法穿戴到此装备位"。

---

## 二、穿戴流程

```
背包右键装备 → 选"穿戴"
  ├── 玩家存活？否 → "你已陨落，无法穿戴"
  ├── equip_slot 匹配装备位？否 → "无法穿戴此物品"
  ├── 装备位为空 → 物品出背包 → atk/def += func_value → 装备位记 item_id → 存档
  └── 装备位已占 → 见替换流程
```

### 装备位为空（示例）
背包有铁剑(4001)，武器位空 → 右键 → 铁剑出背包 → 武器位显示铁剑 → atk 10→13 → 存档。

### 装备位已占（替换）
```
1. 先卸旧装备：旧 item_id 出装备位 → 属性立即恢复（atk/def -= 旧 func_value）
2. 旧装备回背包：
     有空槽 → 占新槽
     背包满 → 替换失败！旧装备保留装备位、新装备保留背包，不做任何更改、不销毁。提示"背包已满，无法替换装备"
3. 旧装备回包成功后 → 穿新装备（同上）→ 存档
```

## 三、属性变化（加算，实时生效）

```
角色 atk = 境界基数(10) + Σ 已穿戴装备 equip_atk
角色 def = 境界基数(5)  + Σ 已穿戴装备 equip_def
```

| 穿戴状态 | 武器位 | 衣服位 | atk | def |
|---------|--------|--------|-----|-----|
| 裸装 | 空 | 空 | 10 | 5 |
| 仅铁剑 | 铁剑 | 空 | **13** | 5 |
| 仅布衣 | 空 | 布衣 | 10 | **7** |
| 双装备 | 铁剑 | 布衣 | **13** | **7** |

- 穿戴/卸下瞬间 atk/def 立即更新，HUD + 角色面板(C键)实时刷新，战斗伤害立即用新值。
- 对小野狼(def2)：裸装 max(10-2,1)=8；穿铁剑 max(13-2,1)=11。
- V0 不做百分比加成（`境界基数×(1+加成%)` 归 V3）。

---

## 四、卸下流程

```
装备位右键已穿戴装备 → 选"卸下"
  ├── 背包有空槽 → 装备回背包 → atk/def -= func_value → 装备位清空 → 存档
  └── 背包满 → 卸下失败，提示"背包已满，无法卸下装备"，装备保留装备位（不销毁）
```

穿卸对称：穿一件+卸同一件 → 属性回归 10/5。

## 五、装备存档（作为角色存档一部分，见 P16）

| 字段 | 类型 | 说明 |
|------|------|------|
| `weapon_item_id` | int | 武器位 item_id，0=空 |
| `armor_item_id` | int | 衣服位 item_id，0=空 |

- 只存 item_id，读档时查 item_config 取 func_type+func_value 重算属性。
- 穿戴/卸下/替换成功时存档；**失败时不存档**（无数据变更）。
- 重进恢复：读 weapon_item_id/armor_item_id → 查物品表 → 应用到 atk/def。退出前穿铁剑+布衣 atk13/def7，重进一致。

## 六、配置表字段（equip_slot_config，2 行）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `slot_id` | int | 1=武器位，2=衣服位 |
| `slot_name` | string | "武器"/"衣服" |
| `slot_key` | string | "WEAPON"/"ARMOR" |
| `accept_type` | enum | 接受的物品 type（WEAPON/ARMOR） |
| `save_field` | string | weapon_item_id / armor_item_id |

```csv
slot_id,slot_name,slot_key,accept_type,save_field
1,武器,WEAPON,WEAPON,weapon_item_id
2,衣服,ARMOR,ARMOR,armor_item_id
```
