# V0 服务器存档字段总表（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P16 ｜ 对应程序 S01/S02
> 融合来源：小马《V0_服务器存档字段总表》(主体,22 字段/version 迁移/CRC/存档时机)。小龙 P16 独立全文本轮未取得，本版以小马为准、未脑补。
> 定位：V0 存档字段的唯一定义（存档"宪法"）。**字段一次定型，version 管迁移，extension_data 预留扩展。命格/道器/轮回仅占位。**

---

## 一、存档架构

| 维度 | 规格 |
|------|------|
| 存储 | 本地 + 云端双写（本地 JSON + KK 平台云同步） |
| 校验 | CRC32（写入时算并记，读取时重校） |
| 版本 | version 字段，结构变更时递增，旧版自动迁移或拒绝 |
| 扩展 | extension_data 字段，V2+ 新增字段进此处，不改基础结构 |
| 粒度 | 按角色（一个 kk_uid 可持多角色 char_id，各自独立存档） |

---

## 二、22 字段定义

| 组 | 字段 | 类型 | 说明 |
|----|------|------|------|
| 标识(4) | kk_uid | string | KK 平台账号 ID |
| | char_id | string | 角色 ID（系统生成） |
| | char_name | string | 角色名 |
| | version | string | 存档版本，V0="v0" |
| 属性(6) | hp / max_hp | int | 当前/最大生命 |
| | mp / max_mp | int | 当前/最大灵力 |
| | atk / def | int | 攻击/防御（含装备加成后的最终值，或基数另算见说明） |
| 成长(5) | realm | string | 境界（V0 固定"练气一层"） |
| | realm_sub | string | 小阶（"初期"） |
| | linggen_id | int | 灵根 ID（1-5） |
| | linggen_name | string | 灵根名 |
| | cult | int | 修为（V0 固定 0） |
| 位置(4) | pos_x / pos_y / pos_z | float | 坐标 |
| | facing | float | 朝向 |
| 背包(1) | backpack | JSON | `[{slot_index,item_id,count}]`（与 P11 一致） |
| 装备(2) | weapon_item_id | int | 武器位 item_id，0=空 |
| | armor_item_id | int | 衣服位 item_id，0=空 |
| 货币(1) | spirit_stone | int | 灵石数量 |
| 占位(3) | mingge_id | int | 命格（V0 占位 0，V2 用） |
| | daoqi_id | int | 道器（V0 占位 0，V5 用） |
| | reincarnation | int | 轮回（V0 占位 0，V6 用） |
| 扩展(1) | extension_data | JSON | V2+ 新增字段容器，V0="{}" |

> atk/def 存档说明：存基数 + 装备 item_id，读档时由 P12 重算最终值；或直接存最终值。程序自选，但须保证重进后与退出前一致。

## 三、version 迁移规则

| 场景 | 处理 | 提示 |
|------|------|------|
| 存档 version < 客户端 | 自动迁移（补缺失字段默认值/转换变更字段/写新 version） | "存档已更新至新版本" |
| 迁移失败 | 保留旧档，建新档 | "存档迁移失败，已为您创建新存档" |
| 存档 version > 客户端 | 拒绝加载（不降级） | "存档版本高于当前游戏版本，请更新客户端" |
| version 非法 | 视为损坏，走修复 | "存档数据异常，正在尝试修复…" |

V0→V2 迁移示例：基础 22 字段不变，extension_data 从 "{}" 初始化为 V2 默认（cult_max/cult_speed 等），version→2。

---

## 四、CRC 校验

写入：构建 JSON(不含 crc)→序列化→算 CRC32→追加 crc 字段→写本地+云。
读取：读 JSON→取 crc→移除后重算→比对，一致则反序列化，不一致走损坏处理（尝试云端备份，失败提示"道基损毁，角色数据异常"）。

---

## 五、存档时机

| 触发 | 写入范围 | 方式 |
|------|---------|------|
| 建角完成 | 全部字段 | 完整写入（初始数据 P14） |
| 属性变化(hp/mp/atk/def) | 属性字段 | 增量（建议 1s 合并窗口） |
| 物品变化(拾取/使用) | 背包+货币 | 增量 |
| 装备变化(穿戴/卸下) | 装备+属性 | 增量 |
| 玩家退出 | 全部字段 | 完整 |
| 定时存档 | 全部字段 | 完整（每 60s 兜底） |
| 房间关闭 | 全部在线角色 | 完整 |

---

## 六、重进恢复

读 KK_UID → 有角色 → 本地优先读（校 CRC）→ 反序列化 22 字段 → 还原角色名/灵根/境界/属性(含装备)/背包/装备位/坐标。坐标恢复到退出位置（非出生点）。

恢复验证（对标 P19 §存档）：角色名/灵根/hp·mp·atk·def/背包(item_id+count+slot)/装备位/坐标 == 退出前。

## 七、与老项目关系

老传奇存档框架（version + extension_data + 本地云双写）**直接复用框架结构**，内容全部置换为修仙字段（属性换 hp/mp/atk/def，无力量敏捷智力/攻击上下限/经验值；经验→修为 cult）。V0 version 从 "v0" 独立起序。
