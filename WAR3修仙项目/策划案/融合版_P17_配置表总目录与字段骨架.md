## 二、加载规则

| 规则 | 说明 |
|------|------|
| 格式 | JSON，UTF-8 无 BOM |
| 时机 | 游戏启动/房间初始化时一次性加载全部 9 表到内存 |
| 读取接口 | `ConfigManager.get_table("xxx_config")` 单例 |
| 字段缺失 | 必填缺失 → **启动报错并阻止进入**（不静默降级） |
| 类型校验 | 加载时校验每字段类型，不符报错 |
| 命名规范 | 表名 `{domain}_config`，字段 snake_case |
| 数组字段 | init_items/slots 等以 JSON 字符串存储 |
| 热更 | V0 不做，代码预留 reload 接口 |

## 三、各表字段骨架

> 完整字段定义见各来源文档，此处给主键与关键字段。

### T1 attr_config（→P02）
`config_id, attr_name, attr_key, attr_type, data_type, parent_attr, formula, display_equip, display_tips, v0_enabled, v0_default, description`。V0 只加载 v0_enabled=true 的 4 行。

### T2 init_char_config（→P14，单行）
`config_id, hp, mp, atk, def, spirit_stone, init_items, spawn_x, spawn_y, facing, realm, realm_sub, cult, linggen_id, linggen_name, bag_size, version`。spirit_stone=100，init_items 不含灵石。

### T3 linggen_config（→P15）
`linggen_id, linggen_name, element, color_hex, icon_id, icon_char, tagline, tooltip, hint, generates, overcomes`。

### T4 monster_config（→P05）
`monster_id, monster_name, level, hp, mp, atk, def, move_speed, atk_range, atk_speed, aggro_range, chase_range, model_id, model_scale, respawn_time, drop_scheme_id`。怪名 小野狼/石傀儡/练气妖兽，monster_id 1001-1003。

### T5 spawn_config（→P07）
`spawn_id, region_id, monster_id, count, max_alive, init_delay, respawn_interval, spawn_type, center_x, center_y, radius`。

### T6 drop_config（→P09）
`scheme_id, scheme_name, drop_way, min_drops, max_drops, slots[{item_id,weight,count_min,count_max,force_drop}]`（32 槽）。

### T7 item_config（→P10）
`item_id, item_name, type, stack_limit, func_type, func_value, equip_slot, icon_id, ground_model, sell_price, quality, level_require, description`。type/func_type/equip_slot 为字符串枚举。item_id 分段 1xxx/2xxx/3xxx/4xxx。

### T8 region_config（→P18）
`region_id, region_name, region_type, left, top, right, bottom`。region_type ∈ safe/spawn/loot_test/boundary。region_id 0 起。

### T9 equip_config（→P12）
`equip_id, equip_name, equip_slot, atk, def, hp, mp, level_require, icon_id, quality, description`。equip_slot ∈ weapon/armor。V0 仅 2 装备位。
