# V0 HUD UI 需求（融合版）

> 版本：vMerge-1.2 ｜ 2026-06-09 ｜ 对应交付清单 P22 ｜ 对应程序 S17 / 美术 A05
> 融合来源：小马《V0_UI需求_HUD》(主体，完整规格 + hud_config.json + 低血闪烁 + 系统提示堆叠) + 小龙《V0_UI需求_HUD》(变体色值/薄修为条取舍点)。
> **本版为两版原稿直接合并**，替代上一版（vMerge-1.1）的"规格推导版"。色值/尺寸以小马原稿为准。
> 定位：屏幕左上角 HUD 规格。WC3 原生黑底灰框，不遮挡战斗视野。

---

## 一、HUD 总则

| 原则 | 规格 |
|------|------|
| 位置 | 屏幕**左上角**固定，不随镜头移动 |
| 风格 | WC3 原生黑底灰框，不做修仙美术化（V1+） |
| 视野 | 不遮挡战斗主视野，面板整体半透明（bg alpha≈60%） |
| 信息层级 | 生命条 > 灵力条 > 修为条(占位) > 角色名/境界 |
| 实时 | 数值条监听属性系统事件刷新，延迟 ≤1 帧(~16ms)，不轮询 |

### V0 显示范围
生命条✅ / 灵力条✅ / 修为条⏳占位 / 角色名✅ / 境界⏳固定"练气一层" / 系统提示区✅ / 灵石数量🟡可选(右上角)。

### V0 不做
小地图 / 任务追踪 / 技能栏快捷栏 / 队友血条队伍面板 / Buff图标 / 聊天框 / 经验条(修仙无经验,用修为替代) / 目标血条(怪物血条由怪物系统挂头顶,不在本文档)。

## 三、生命条（HP Bar）

| 属性 | 规格 |
|------|------|
| 标签 | `HP`（白色粗体 10px，左侧宽 24px） |
| 正常填充色 | **`#FF3333`**（红） |
| 未填充底色 | `#333333` |
| 长×高 | 160×12 px |
| 边框 | 1px `#555555` |
| 填充 | 从左到右，`fill = hp_cur/hp_max` |
| 数值 | 条右侧 `{当前}/{最大}`（如 100/100） |

### 低血量警告（小马真功能，务必保留）
- 触发：`hp_cur/hp_max < 0.20`。
- 效果：血条 **2Hz 闪烁**（红 `#FF3333` ↔ 暗红 `#991111`，周期 500ms）；数值文字同步变红闪烁。
- 停止：血量回到 ≥20% 或角色死亡（死亡立即停闪、血条清空）。

### 更新时机
受击扣血 / 用回复物品 / 复活回满 / 读档初始化。监听属性系统 `on_attr_change`，不轮询。V0 不做平滑缓动（数值直接跳变）。

---

## 四、灵力条（MP Bar）

| 属性 | 规格 |
|------|------|
| 标签 | `MP`（白色粗体 10px，左侧宽 24px） |
| 填充色 | **`#3366FF`**（蓝） |
| 未填充底色 | `#333333` |
| 长×高 | 160×12 px（与血条对齐） |
| 边框 | 1px `#555555` |
| 数值 | 条右侧 `{当前}/{最大}`（如 50/50） |
| 低灵力警告 | <20% 时 2Hz 闪烁（蓝 `#3366FF` ↔ 暗蓝 `#1A3380`） |

> V0 无技能耗蓝，mp 仅由回灵丹/灵泉水改变，低灵力闪烁极少触发，但规则先定义好。

## 五、修为条（占位）

| 属性 | 规格 |
|------|------|
| 标签 | `修为`（灰 `#888888` 10px，左侧宽 32px） |
| 填充色 | `#555555`（灰，固定不变） |
| 底色 | `#333333`（与条色刻意低对比，视觉上"未激活"） |
| 长×高 | 160×12 px |
| 填充 | 始终 0%（固定空条，不增长） |
| 数值/标注 | 条右侧 `V2开放`（灰 `#888888` 9px） |
| 交互 | 不可点；**悬停 Tooltip**：「修为系统将在 V2 版本开放，届时可通过打坐修炼积累修为、突破境界」 |

### V2 预留（记录，V0 不实现）
V2 启用后：颜色 `#9933FF`（紫）、水平填充、数值 `{当前修为}/{突破所需}`、满条时边框金色闪烁提示可突破。

## 六、角色名与境界

| 元素 | 规格 |
|------|------|
| 角色名 | 白 `#FFFFFF` 12px 粗体，第一行左侧，最多 8 中文字符（超出截断+"…"），读存档 `char_name` |
| 境界 | 浅灰 `#AAAAAA` 11px 常规，角色名右侧间距 4px，V0 硬编码"练气一层"，V2 后读 `realm`+`realm_sub` |

第一行效果：`玩家名称  练气一层`（白 + 浅灰）。

---

## 七、系统提示区（与 P24 协同）

| 属性 | 规格 |
|------|------|
| 位置 | 屏幕**中央偏上**，水平居中，垂直约屏高 25% 处 |
| 文字 | 白 `#FFFFFF` 11px，**1px 黑色描边**（任何场景可读，描边=底层黑字偏移1px+上层白字） |
| 背景 | 无独立背景，文字直接叠加画面 |
| 持续 | 3 秒（前 2s 不透明，最后 1s alpha 255→0 线性渐隐） |
| 堆叠 | 向上堆叠，新提示在上，每条间距 20px，**最多同时 5 条**，超出最早一条立即消失（FIFO） |
| 文案 | 全部走 P24（G7 锁定格式），如"道基已固（保存成功）"、"储物袋已满（背包已满）"、"获得 灵石 ×2" |

> 伤害飘字（白色头顶上浮 1.5s）属战斗表现，规格见 P03 §4，不在系统提示区。

## 八、其他显示

### 灵石数量（右上角，🟡可选）
右上角距边 16px，灵石图标(金菱形 16×16)+数值，金色 `#FFD700` 12px，格式 `◇ 0`，监听货币变化。美术来不及可用金色圆点 `●` 占位。V0 可不做。

### 死亡复活面板（属战斗 UI，非 HUD 常驻）
hp≤0 → 屏幕正中"你已陨落" → 3s 后"回城复活"按钮。详见 P03 §3.3。
