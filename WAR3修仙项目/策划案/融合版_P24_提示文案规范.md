# V0 提示文案规范（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P24 ｜ 对应程序 S19
> 融合来源：小马《V0_提示文案规范》(主体,PROMPT_ID 枚举/ID码/全规格) + 小龙《V0 提示文案规范》(分类参考)。
> 定位：V0 全部系统提示的统一文案与展示规格。**文案格式锁定：修仙化主文案 +（括号直白），如"道基已固（保存成功）"。**

---

## 一、文案总则

- 格式锁定：`修仙化主文案（直白化）`。修仙化保代入感，括号直白保灰盒测试可读。
- 程序用 PROMPT_ID 枚举调用，不在代码里写死字符串。
- 每条文案带：ID 码 / 文案 / 类型 / 颜色 / 时长 / 位置。

---

## 二、PROMPT_ID 分段编码

| 段 | 范围 | 类别 |
|----|------|------|
| A | A01-A99 | 存档/系统 |
| B | B01-B99 | 战斗 |
| C | C01-C99 | 物品/背包 |
| D | D01-D99 | 角色/创角 |
| E | E01-E99 | 区域/通用 |

## 四、展示规格

| 项目 | 规格 |
|------|------|
| 系统提示位置 | 屏幕中上偏上，不挡 HUD/战斗视野 |
| 飘字位置 | 单位头顶 (0,+60)，上浮 40px |
| 颜色 | 白=常规，红=错误/严重，黄=警告/受阻，绿=恢复，灰=占位/消失 |
| 动画 | 淡入(0.2s)→停留→淡出(0.5s)；飘字上浮+渐隐 1.5s |
| 堆叠 | 多条系统提示纵向堆叠，旧的上移渐隐 |
| 字号 | 系统提示 14px，飘字 12px |
| 占位符 | `{物品名}/{数量}/{伤害数值}/{数值}` 运行时替换 |

## 五、程序调用

```
ShowPrompt(PROMPT_ID.A01)                  // "道基已固（保存成功）"
ShowPrompt(PROMPT_ID.C01, {item:"灵石", count:2})  // "获得 灵石 ×2"
ShowFloatingText(unit, PROMPT_ID.B03, {dmg:8})     // 头顶白字"8"
```
所有文案集中在 prompt_config（枚举表），程序不硬编码字符串，便于改文案/多语言。
