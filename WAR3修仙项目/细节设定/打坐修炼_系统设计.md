练气一层 · 前期      练气四层 · 中期      练气七层 · 后期

## 二、修炼速度公式

```
每3秒获得修为 = 境界基础速率(3秒) × (1 + A类加成) × (1 + B类加成) + Σ(固定值)
```

即时修为不经过速率公式：

```
修为 += effect_value  （吃丹时当场查当前境界列，直接累加）
```

### A 类 vs B 类

| | A 类 | B 类 |
|------|------|------|
| 是什么 | 先天稀有源，独立乘区 | 常规系统源，汇总乘区 |
| 数量 | 极少 | 几乎所有加成系统 |
| 值钱逻辑 | 有就是质变 | 堆积竞争 |
| 跨境界 | 永久不衰减 | 查 cult_effect_config 取当前境界系数 |
| 例子 | 天灵根、先天道器 | 丹药buff、洞府灵气浓度、功法加成、阵法、装备词条、聚灵阵、时段灵气… |

### 固定值

所有固定值来源直接累加，不参与乘法。查配置表取当前境界值。

## 三、核心原则：不存值，只存来源

CultBonusManager **不缓存任何来源的具体数值**。它只记录：

- 当前有哪些来源（source_id）
- 每个来源有几个实例/堆叠数
- 限时来源的剩余时间

**每次计算修炼速度时：**

1. 遍历所有活跃来源
2. 取玩家当前境界（realm + realm_level）
3. 当场查 `cult_effect_config` 表，取每个来源在当前境界的 `effect_value`
4. 同类汇总后代入公式

这意味着：
- 一颗丹药在筑基期吃的 → 到了金丹期自动按金丹行取值（不存死筑基期的数）
- 一颗洗髓丹（永久）→ 永远在来源池里，每次 recalc 都重新读表
- A 类源（天灵根）也是一样：永久在池子里，每次查表。表里全境界同值，所以表现上"不衰减"

### 堆叠

同 item_id 的限时来源可以有多个实例（吃了 3 颗丹药 = 3 个独立倒计时）。recalc 时各自读表取值后累加。永久源同 source_id 只保留一个（吃药覆盖）。

## 五、丹药效果：即时修为 + 速率 buff 可并存

一颗丹药可同时有即时修为和速率 buff。

```
item_id | effect_dim    | bonus_type | duration | 练气  | 筑基  | 金丹  | …
培元丹   | instant_cult  | —          | 0        | 500   | 200   | 50    | …
培元丹   | speed_buff    | b_bonus    | 1800     | 0.20  | 0.05  | 0.00  | …
```

吃培元丹：
1. 当场查表取当前境界的 instant_cult 值 → 修为累加
2. 往 CultBonusManager 注册一个 B 类限时源（item_id=培元丹, pool=b, duration=1800）

丹药不能无限吃——**丹毒**累积，超过阈值触发负面效果。丹毒机制（丹毒值/阈值/负面效果/排出速率）**挂待决**。

## 六、功法修炼方式

修为只来自打坐（在线/离线）。功法各管各的方向：

| 功法方向 | 加成目标 | 归属系统 |
|---------|---------|---------|
| 修炼功法 | 修炼速度（B 类） | 打坐修炼 L |
| 剑道功法 | 攻击力/暴击/技能伤害 | 战斗 C |
| 丹道功法 | 炼丹成功率/品质/丹毒抗性 | 炼丹 S |
| 阵道功法 | 阵法效果/神识探测 | 阵法 V |
| 兽道功法 | 灵兽培养/驾驭 | 灵兽 AK |

不做"每个功法都加修为"。理由：与长老服务体系一致（所有玩家走所有长老路子刷好感），功法各管各的 = build 自由组合。

## 八、在线打坐交互

- 进入打坐：宗门修炼室或洞府闭关位交互触发
- 打坐期间不能战斗、不能移动
- 每 3 秒一跳，修为自动累加
- UI：进度条 + 速率数值
- 修为满后出现突破按钮（如果 need_breakthrough=true 则跳转到突破流程）
- 随时可中断，无惩罚
- 大境界突破：修为满 + 材料齐 → 渡劫台 → 渡劫事件

| 大境界 | need_breakthrough | 说明 |
|--------|:--:|------|

## 十、修炼加成管理器 CultBonusManager

属性管理器的子系统插件。

```
属性管理器（总管）
    ├── 基础属性：hp, mp, atk, def
    ├── 修炼属性：cult, cult_max, cult_speed ← 委托给 CultBonusManager
    ├── 五行属性
    └── 境界属性：realm, realm_sub, ...

CultBonusManager
    ├── 来源池（只存来源身份，不存值）
    │   ├── A 类池：{source_id → [instance]}     ← 天灵根、先天道器
    │   ├── B 类池：{source_id → [instance]}     ← 丹药/功法/洞府/灵气浓度…
    │   └── 固定值池：{source_id → [instance]}   ← 装备词条/flat 丹药
    │
    │   每个 instance = {remaining_seconds}  (0=永久)
    │
    ├── 缓存：dirty 标记 + cached_speed
    │
    ├── 来源操作
    │   ├── add(item_id, pool, duration)
    │   │     往对应池子加一个实例。同 item_id 可多实例（堆叠）
    │   ├── remove(item_id, instance_id)
    │   └── mark_dirty()                    ← 境界变化/装备更换时调用
    │
    ├── 计算（每次调用当场读表）
    │   └── recalc():
    │         遍历所有活跃实例 → 取当前境界 → 查 cult_effect_config
    │         → ΣA / ΣB / Σflat → 代公式 → 缓存
    │
    └── 对外
        ├── get_cult_speed()           ← 自动 recalc if dirty
        └── preview(...)               ← 模拟计算，不改实际状态
```

**recalc 流程：**

```
for each 活跃实例 in (A池 + B池 + flat池):
    value = cult_effect_config.lookup(item_id, player.realm).effect_value
    if instance 属于 A池: ΣA += value
    if instance 属于 B池: ΣB += value
    if instance 属于 flat池: Σflat += value

speed = base_rate × (1 + ΣA) × (1 + ΣB) + Σflat
```

- 同 item_id 的多个实例各自读表（值相同但 timer 不同）
- 境界变化 → mark_dirty() → 下次 get_cult_speed() 自动重算
- 任何系统只需调用 `add()`，不需要知道公式细节

## 十一、配置表

### realm_config（小境界配置）

80 行。

| 字段 | 类型 | 说明 |
|------|------|------|
| realm | enum | 大境界 |
| realm_level | int | 1-10 |
| stage_label | string | 前期/中期/后期/圆满 |
| cult_required | int | 突破所需修为（圆满行=大境界突破阈值） |
| base_cult_speed | float | 该层基础速率（修为/3秒） |
| base_hp | int | 基础生命值 |
| base_mp | int | 基础灵力值 |
| base_atk | int | 基础攻击 |
| base_def | int | 基础防御 |
| need_breakthrough | bool | 升到下一层是否需要突破事件 |

小境界同时做两件事：横向打开容器（法力扩容/神识增强→法宝位/术法品阶等）+ 给属性数值。大境界做数量级跃迁。

### cult_effect_config（物品效果配置）

行 = 物品 × 效果维度，列 = 境界。

| 字段 | 类型 | 说明 |
|------|------|------|
| item_id | int | 物品/效果 ID |
| effect_dim | enum | instant_cult / speed_buff |
| bonus_type | enum | a_bonus / b_bonus / flat（instant_cult 留空） |
| duration | int | 秒，0=永久（instant_cult 固定 0） |
| value_练气 ~ value_渡劫 | float | 8 列，各境界系数 |

### breakthrough_config（突破事件配置）

| 字段 | 类型 | 说明 |
|------|------|------|
| realm | enum | 大境界 |
| realm_level | int | 层数 |
| required_items | json | 所需道具 [{item_id, count}, ...] |
| required_spirit_stones | int | 所需灵石 |
| success_rate | float | 基础成功率 0-1 |
| fail_penalty_cult_pct | float | 失败扣除修为比例 |
| fail_debuff_id | int | 失败 debuff ID |
| guaranteed_after_fails | int | 保底次数（连续失败 N 次后必成） |

### area_qi_density（区域灵气浓度）

| 字段 | 类型 | 说明 |
|------|------|------|
| area_id | int | 区域 ID |
| area_name | string | 区域名 |
| qi_bonus | float | B 类加成值 |

### root_bonus（灵根→A 类加成）

| 字段 | 类型 | 说明 |
|------|------|------|
| root_type | enum | 天灵根/真灵根/双灵根/三灵根/杂灵根 |
| purity_min | int | 纯度下限 |
| purity_max | int | 纯度上限 |
| a_bonus | float | A 类加成值 |
