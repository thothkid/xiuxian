## 一、枚举表是什么

枚举表（`__enums__.xlsx`）定义游戏中共用的枚举值——同类物品共享的属性分类，如颜色、品质、装备位置等。具体表里填枚举的 name（如 `Green`），程序通过 name 映射到对应的 value。

## 二、表结构

### 枚举级字段（A-G列，一个枚举类型占多行，首行填枚举信息）

| 列 | 字段 | 说明 |
|---|---|---|
| A | full_name | 枚举全名（如 `ItemColor`） |
| B | flags | 是否为位标记枚举（如 `System.IO.FileMode`，填数据时可为 `READ\|WRITE` 这种组合） |
| C | unique | 枚举项是否唯一（`True`/`False`） |
| D | group | 分组标识 |
| E | comment | 枚举类型注释（F列？TODO:确认列名对应关系） |
| F | tags | 标签 |

### 枚举项字段（H列起，每个枚举值一行，首行留空）

| 列 | 字段 | 说明 |
|---|---|---|
| H | name | 枚举名（代码标识符，如 `White`、`Green`、`Head`） |
| I | alias | 别名（中文显示名，如"白色""绿色""头部"）。别名和注释仅枚举表有，具体数据表不重复 |
| J | value | 枚举值（int，从 1 开始，不重复） |
| K | comment | 该项的注释说明 |
| L | tags | 标签 |
| M | table | 关联的外部表名（如多语言表时填 `多语言.xlsx`） |
| N | key | 外部表的键字段名 |
| O | value | 外部表的值字段名 |
| P | comment | 外部关联说明 |

## 三、当前已定义的枚举

## 四、使用方式

- **枚举表**：定义 name、alias、value、comment，alias 和 comment 仅在此处维护
- **具体数据表**：填 name（如 `Green`），程序通过枚举表查到对应的 value=2、alias="绿色"
- **Table 列**：暂不用，以后多语言等场景使用

特殊说明：

bool： true、false、0、1、是、否 都是有效值。另外大小写不敏感，如True也是合法bool值。填其他值如abc、4则会发生解析错误
string：单元格留空即为长度为0的字符串。但读取流式格式时，空白单元格也会被当作无用单元格而被忽略，此时可以用""表示长度为0的字符串。string默认不处理转义， 如果希望将字符串中\n替换为换行符，则需要添加上escape=1tag，如string#escape=1。

enum 数据格式
可以填写枚举项的变量名、别名或者对应的整数。如果是flags类型枚举，还可以填'A|B'这样的或形式枚举。flags类型enum还支持列限定模式， 让每个枚举值占一列，然后将包含的标志位列标为1，表示最终的枚举值包含此项。

enum

如果enum中有值为0的枚举项，则可以留空，自动取该枚举项，否则会解析出错。

对于flags类型的enum，允许填多个枚举值来表示组合，默认使用'|'来分割多个枚举值。可以通过设置enum的sep属性来指定其他分割符。如sep=","时，使用A,B表示两个枚举值的或组合。

除了容器以外的类型都可以是可空类型。所有可空类型都可以用null表达空值。
