# 枚举表配置说明

> 依据当前 `__enums__.xlsx` 实际结构整理，2026-07-10 更新。
>
> 本文只描述当前项目的枚举配置格式；以 `__enums__.xlsx` 和 `__beans__.xlsx` 为最终准则。

## 1. 枚举的作用

枚举用于约束一组固定分类值。业务表可填写枚举项的：

- `name`：代码名，例如 `White`、`ByProbability`、`Attr001`；
- `alias`：中文/策划名，例如 `白色`、`掉落方式_按概率`、`生命值`；
- `value`：整数值，例如 `1`、`2`。

导表时三种写法都会映射到同一枚举值。优先在业务表中填写清晰、稳定的 `alias`，不要依赖数值记忆。

## 2. `__enums__.xlsx` 的实际列结构

`Sheet1` 前三行是字段定义和说明，数据从第 4 行开始。

### 2.1 枚举类型字段

一个枚举类型可以占多行；仅首行填写类型字段，后续枚举项行留空即可。

| 列 | 字段 | 填写规则 |
|---|---|---|
| A | 标记列 | 固定为表头/说明标记，数据行不填 |
| B | `full_name` | 枚举类型名，例如 `ItemQuality` |
| C | `flags` | 是否位标记枚举；当前所有枚举均为 `false` |
| D | `unique` | 枚举数值是否唯一；当前均为 `true` |
| E | `group` | 枚举分组，可留空 |
| F | `comment` | 枚举类型说明 |
| G | `tags` | 标签，可留空 |

### 2.2 枚举项字段

| 列 | 字段 | 填写规则 |
|---|---|---|
| H | `name` | 稳定的代码标识符；同一枚举内不可重复 |
| I | `alias` | 业务表常填的中文/策划名；同一枚举内不可重复 |
| J | `value` | 整数；`unique=true` 时不可重复 |
| K | `comment` | 枚举项说明 |
| L | `tags` | 标签，可留空 |
| M-P | 外部表映射 | `table.name / key / value / comment`；当前项目未启用 |

| 枚举 | 用途 | 典型 alias |

| `DropType` | 掉落方案方式 | 掉落方式_按概率 |
| `AttrRole` | 属性配置角色 | BASE、MODIFIER |
| `InfluenceOp` | 属性影响运算 | 无、ADD、MUL |
| `EquipPart` | 装备部位 | 冠、法袍、武器、佩/玉佩、印绶/符印、鞋履 |
| `EquipRank` | 装备阶级 | A新手、B沃玛级、C祖玛级、D苍月级、E赤月级 |
| `MonsterDignity` | 怪物身份 | 普通、精英、BOSS、独特 |
| `ItemDisplayType` | 物品显示分类 | 强化道具、特戒、技能书、货币、外观、药品 |
| `AttrKey` | 属性键 | 无、生命值、灵力值、攻击下限等 |

## 4. 业务表中如何使用

1. 看业务表第 2 行 `##type`。字段类型等于枚举名时，该列必须填写对应枚举值。
2. 允许填写该枚举的 `name`、`alias` 或整数 `value`；推荐填写 `alias`。
3. `array,Attr`、`array,AttrModifier` 等 Bean 容器中的属性名同样受 `AttrKey` 约束。
4. `flags=true` 的枚举才允许使用 `A|B` 组合写法；当前项目没有启用 flags 枚举。
5. 除枚举值为 `0` 且被定义为默认项的情况外，不要用空单元格替代枚举值。

示例：

| 业务字段类型 | 推荐填写 | 也可填写 |
|---|---|---|
| `ItemQuality` | `白色` | `White`、`1` |
| `DropType` | `掉落方式_按概率` | `ByProbability`、`1` |
| `AttrKey` | `攻击上限` | `Attr004`、`4` |
| `InfluenceOp` | `MUL` | `Mul`、`2` |

## 5. 新增或修改枚举的规则

1. 先确认是否确属全局共用分类；仅单张表使用的自由文本不要强行建枚举。
2. 在 `__enums__.xlsx` 追加枚举项，不要修改已上线项的 `name` 或 `value`。
3. `alias` 可以改文案，但会影响所有直接填写 alias 的业务表；改前必须全表检索。
4. 新枚举必须至少有一个枚举项；值从 `1` 开始连续递增更便于维护。
5. 若需要“无/默认”值，显式新增 `value=0` 项，并确认业务字段允许留空取该默认值。
6. 新增后检查 `__beans__.xlsx`、各业务表的 `##type` 和实际数据值是否一致。

## 6. 与导表格式的关系

- 业务数据 Sheet 的 A1 必须以 `##` 开头，通常第 1 行为 `##var`、第 2 行为 `##type`、第 3 行为 `##group`。
- `__enums__.xlsx`、`__beans__.xlsx`、`__tables__.xlsx` 是 schema 文件，由 `luban.conf` 注册。
- 业务表只有被 `__tables__.xlsx` 登记后才会导出；未登记的工作簿不会自动成为运行时配置。

## 7. 当前注意事项

- 旧版说明中的 `ItemColor`、`LangDefines` 并非当前 `__enums__.xlsx` 中的枚举；请不要按旧名称新配数据。
- 当前外部表映射列（M-P）未使用。多语言表若要接入，需要先补充对应枚举和外部映射定义，再登记到导表流程。
- 枚举配置完成后，应执行跨表校验，至少检查：枚举值、主键唯一性、怪物/技能/刷怪方案等 ID 引用。
