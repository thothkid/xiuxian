## 〇、程序写属性系统前必读（最高优先级）

### V0 只需关注这 4 个字段

| 程序字段 | 中文名 | 初始值(练气一层) | 类型 |
|----------|--------|-----------------|------|
| `hp` | 生命值 | 100 | int |
| `mp` | 灵力值 | 50 | int |
| `atk` | 攻击 | 10 | int |
| `def` | 防御 | 5 | int |

### 三条红线（踩了必返工）

1. **严禁力量/敏捷/智力**：修仙无三围。老传奇 ConfigAttr 的读表框架可复用，但属性名全部替换。S07 验收已写明一票否决项。
2. **攻击/防御是单值**：不是 `atk_min/atk_max`、不是 `物防/魔防` 二元结构。
3. **魔法值→灵力值（仅改名）**：蓝条仍是一条数值条，程序字段保持 `mp`（兼容老代码），中文名必须是"灵力值"。

### 推荐做法

```
1. 复用老 ConfigAttr 加载/读取框架
2. attr_config 表换成本文档第七章结构
3. V0 只读 v0_enabled=true 的 4 行，其余 52 行字段预留但跳过
4. 属性对外接口统一用 attr_key("hp"/"mp"/"atk"/"def")，不用 config_id 硬编码
5. 计算公式先硬编码(V0 只有 max(atk-def,1))，后续版本再读 formula 字段
```

## 一、设计依据

### 1.1 核心公式框架（V2+ 启用，V0 不实现百分比）

```
实际属性 = 境界基数 × (1 + 装备加成%) + 装备固定值
```

V0 简化为：`实际属性 = 境界基数 + Σ 装备固定值`。百分比加成归 V3。

### 1.2 境界 vs 装备分工

| 维度 | 境界给（下限/潜力） | 装备给（上限/变现） |
|------|-------------------|-------------------|
| 攻防生命 | 基础基数，大境界数量级跃迁 | 加成% + 固定值 |
| 暴击/攻速/施法 | ❌ 不给 | 纯装备区 |
| 特殊机制 | 境界压制(PvE/PvP) | 吸血/反弹/护盾/触发 |
| 修炼 | 打坐效率基数 | 灵气加成/修炼加速 |
| 区域权限 | 解锁新地图 | ❌ 不给 |

大境界碾压：筑基裸装 > 练气毕业装；同境界内卷靠装备拉 3-5 倍。

## 二、属性分类总览（56 属性，六大类）

### 第一类 基础生存（BASE，V0 实现）

| ID | 属性 | 字段 | 类型 | V0 |
|----|------|------|------|-----|
| 1 | 生命值 | hp | INT | ✅ |
| 2 | 灵力值 | mp | INT | ✅ |
| 3 | 攻击 | atk | INT | ✅ |
| 4 | 防御 | def | INT | ✅ |

### 第二类 战斗（COMBAT，V1-V3 启用，V0 字段预留）

术法攻击 matk(10)、暴击率 crit(11)、暴击伤害 critdmg(12)、攻击速度 aspd(13)、施法速度(14)、冷却缩减(15)、命中 hit(16)、闪避 dodge(17)、抗性 resist(18)。

### 第三类 修炼成长（CULT，V2 启用，V0 仅修为条占位）

修为 cult(20)、修为上限 cult_max(21)、修炼速度 cult_speed(22)、灵气亲和 qi_affinity(23)、突破加成 breakthrough(24)。

### 第四类 五行与灵根（FIVE，V2 启用）

五行亲和 gold/wood/water/fire/earth(30-34)、灵根纯度 root_purity(35)。

### 第五类 境界（REALM，V2 启用，V0 固定练气一层）

当前境界 realm(40)、当前小阶 realm_sub(41)、境界压制PvE(42)、境界压制PvP(43)、破压制 anti_suppress(44)。

### 第六类 特殊（SPEC，V5-V6 启用）

神识 divine_sense(50)、神识范围 sense_range(51)、法宝出战位 artifact_slots(52)、丹毒 pill_toxin(53)、丹毒抗性 toxin_resist(54)、寿元 lifespan(55)、道心 dao_heart(56)。

## 五、配置表字段定义（attr_config）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `config_id` | int | ✅ | 属性编号 1-56，主键 |
| `attr_name` | string | ✅ | 中文名 |
| `attr_key` | string | ✅ | 程序键名（程序读此字段，勿用 config_id 硬编码） |
| `attr_type` | enum | ✅ | BASE/COMBAT/CULT/FIVE/REALM/SPEC |
| `data_type` | enum | ✅ | INT/FLOAT/PERCENT/ENUM |
| `parent_attr` | string | ❌ | 父级属性 Key（V2+ 计算链用） |
| `formula` | string | ❌ | 计算公式（V0 不读，V2+ 启用） |
| `display_equip` | bool | ✅ | 是否显示在装备栏 |
| `display_tips` | bool | ✅ | 是否显示在 Tooltip |
| `v0_enabled` | bool | ✅ | **V0 是否生效（最关键）** — 程序仅加载 true 的行 |
| `v0_default` | string | ❌ | V0 默认值（v0_enabled=true 时必填） |
| `description` | string | ❌ | 说明文本 |

### V0 四属性 CSV 示例

```csv
config_id,attr_name,attr_key,attr_type,data_type,parent_attr,formula,display_equip,display_tips,v0_enabled,v0_default,description
1,生命值,hp,BASE,INT,realm_hp_base,realm_hp_base*(1+equip_hp_pct)+equip_hp_fixed,true,true,true,100,角色生存核心。归零=死亡。大境界跃迁倍增
2,灵力值,mp,BASE,INT,realm_mp_base,realm_mp_base*(1+equip_mp_pct)+equip_mp_fixed,true,true,true,50,施法消耗资源。修士体内可调用的灵气总量
3,攻击,atk,BASE,INT,realm_atk_base,realm_atk_base*(1+equip_atk_pct)+equip_atk_fixed,true,true,true,10,物理伤害输出基础值。修仙用单值替代传奇上下限
4,防御,def,BASE,INT,realm_def_base,realm_def_base*(1+equip_def_pct)+equip_def_fixed,true,true,true,5,物理伤害减免基础值。修仙用单值替代传奇上下限
```
