## 一、伤害公式

```
damage = max(攻击方.atk - 防守方.def, 1)
```

- **硬编码**，不要写成可配置公式字符串。V0 不需要公式解析器。
- 必中、无暴击、无闪避、无抗性、无元素克制。
- 保底伤害 1：攻 ≤ 防时仍造成 1 点（不出 0 或负数）。
- 属性读取走单位接口 `GetAttr(unit,"atk")` / `GetAttr(unit,"def")`，不直接读配置表或全局变量。

## 二、攻击流程与节奏

### 2.1 单次攻击周期

```
前摇 0.30s → 命中帧(伤害计算+飘字) → 后摇 0.40s → 冷却 1.50s → 下一轮
完整周期 = 2.20s
```

- 前摇结束触发伤害回调；后摇结束触发冷却开始。用独立 Timer 精确控制，不在动画回调里做战斗逻辑。
- 攻击范围（近战）：128 码（来自 combat_config.attack_range_melee）。范围外先移动到范围内再起手。
- 冷却独立计算：交替攻击不同目标时各自冷却独立，切换目标不跳过冷却。

### 2.2 攻击指令

选中怪物 → 右键/A 键 → 自动走到攻击范围 → 攻击循环。

## 三、死亡与复活

### 3.1 死亡判定优先级

```
每次扣血后立即检查 hp <= 0
同一帧内多次攻击：按发起顺序依次扣血，死亡当帧后续攻击全部失效（不可鞭尸）
```

### 3.2 玩家死亡 → 复活

- hp ≤ 0 → 播放死亡 → 3 秒后弹"回城复活"按钮。
- 点击复活：坐标重置为出生点（区域表出生区中心）、hp=max_hp、mp=max_mp、移除死亡标记、恢复碰撞。

### 3.3 怪物死亡

| 项目 | 规则 |
|------|------|
| 死亡触发 | hp ≤ 0 当帧 |
| 死亡动画 | 约 1.0s |
| 掉落触发 | 死亡动画播放时（0.3s 内）触发掉落系统 |
| 尸体保留 | 动画结束后保留 5s（可配置），期间不可选中/不可攻击 |
| 尸体消失 | 5s 后约 0.5s 淡出 |
| 触发重刷 | 尸体消失后刷新点死亡计数+1，重刷倒计时开始（重刷间隔见 P07/P05） |

## 四、飘字规则

### 4.1 伤害飘字

| 属性 | 规则 |
|------|------|
| 内容 | 伤害数值整数（如 "8"） |
| 颜色 | 白色 RGB(255,255,255) |
| 时机 | 伤害计算完成、扣血生效时 |
| 位置 | 受击单位头顶 (0,+60) 像素 |
| 运动 | 向上匀速 ~40 像素，同时渐隐 |
| 时长 | 1.5s（前 0.5s 不透明，后 1.0s alpha 255→0） |
| 字号 | 12px |
| 叠加 | 多次伤害独立叠加，每次 +15 像素横向错开 |

### 4.2 V0 不做的飘字

暴击(黄字大字)/闪避/免疫/吸收/治疗(绿字)/技能名/境界压制——全部 V1-V4，V0 只有白色伤害数字。

### 4.3 WC3 实现建议

`CreateTextTag()` → `SetTextTagText(tag,"-"+I2S(dmg),0.012)` → `SetTextTagColor(255,255,255,255)` → `SetTextTagVelocity(0,0.03)` → 1.5s 后 `DestroyTextTag()`，0.5-1.5s 间逐步降 alpha。

## 五、战斗数值示例（初始玩家 vs 三测试怪）

> 玩家练气一层：hp=100, atk=10, def=5。怪物数值见 P05。

| 场景 | 公式 | 单次伤害 | 击杀次数 | 耗时(2.2s/次) |
|------|------|---------|---------|--------------|
| 玩家→小野狼 | max(10-2,1) | 8 | 4 | ~9s |
| 小野狼→玩家 | max(6-5,1) | 1(保底) | 100 | — |
| 玩家→石傀儡 | max(10-4,1) | 6 | 10 | ~22s |
| 石傀儡→玩家 | max(8-5,1) | 3 | 34 | ~75s |
| 玩家→练气妖兽 | max(10-3,1) | 7 | 12 | ~26s |
| 练气妖兽→玩家 | max(11-5,1) | 6 | 17 | ~37s |

> 练气妖兽 37s 可杀死玩家，玩家 26s 杀它——略占优但必须主动进攻，站着不动会死，用于验证死亡→复活闭环。

## 六、配置表字段（combat_config）

| 字段名 | 类型 | 建议值 | 说明 |
|--------|------|--------|------|
| attack_range_melee | float | 128.0 | 近战攻击范围（码） |
| attack_windup | float | 0.30 | 前摇（秒） |
| attack_backswing | float | 0.40 | 后摇（秒） |
| attack_cooldown_base | float | 1.50 | 攻击冷却（秒，不含前后摇） |
| corpse_duration | float | 5.0 | 尸体保留（秒） |
| death_anim_duration | float | 1.0 | 死亡动画（秒） |
| revive_delay | float | 3.0 | 玩家死亡后复活按钮延迟（秒） |
| damage_floor | int | 1 | 保底伤害 |

> 战斗配置在游戏初始化时从 `combat_config.json` 加载，运行时用缓存值，不每次攻击读文件。

## 七、给程序的关键提醒

1. 伤害公式硬编码 `max(atk-def,1)`，不写公式字符串。
2. 属性走 `get_attr(unit,"atk"/"def")` 接口。
3. 扣血后立即判死，同帧不鞭尸。
4. 前后摇用独立 Timer，不在动画回调里做战斗逻辑。
5. 每个飘字独立生命周期 Timer，不用全局队列。
6. 怪物死亡→掉落→刷新三段由战斗系统发事件，掉落/刷新各自监听，不硬耦合。
7. 严禁旧传奇属性名（DC/AC/MC/SC），V0 属性是 atk/def/hp/mp。
