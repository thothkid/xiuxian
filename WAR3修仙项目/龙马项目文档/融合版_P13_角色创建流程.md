# V0 角色创建流程（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P13 ｜ 对应程序 S05/S16
> 融合来源：小马《V0_角色创建流程》(双端伪代码/错误码/幂等/时序主体) + 小龙《V0 角色创建流程》(流程图导览并入)。
> **角色名规则按锁定裁决：min=1 / 禁纯数字·纯空格·特殊符号 / V0 敏感词关。** 覆盖小马原稿的 min=2、敏感词开。

---

## 一、创角触发

```
玩家进房(KK入口) → 服务器查 KK_UID
  ├── 已有角色 → 跳过创角，读存档直接进游戏
  └── 无角色 → 打开创角界面，强制完成创角
创角中途退出（未建档）→ 下次仍视为首次，重走创角
```

三条铁律：①必选灵根（不可跳过/不可随机，未选则确认按钮灰显）②KK_UID 唯一（不查重名，KK_UID 保证唯一）③建档幂等（同 KK_UID 重复请求 = 返回已有角色）。

V0 不做：性别选择(V2)、模型预览(V0 占位)、命格选择(V2)、随机属性/骰点(全固定)、新手引导(V1)。

---

## 二、创角流程

```
步骤1 命名 → 输入角色名 → 客户端校验(见§三) → 通过
步骤2 灵根 → 展示金木水火土(全未选) → 点击选中高亮 → 5选1互斥
步骤3 确认 → 显示摘要(名+灵根) → 点"踏入修仙"
          ├── 未输名 → "角色名不能为空" → 回步骤1
          ├── 未选灵根 → "请先选择灵根" → 回步骤2
          └── 通过 → 提交创建请求
步骤4 建档 → 服务器校验KK_UID无角色 → 生成角色ID → 写初始数据(P14) → 写存档
          ├── 成功 → 返回角色数据 → 步骤5
          └── 失败 → 显示错误 → 可重试
步骤5 进入 → 加载角色 → 出生(0,0) → 完成
```

## 三、角色名校验【锁定规则】

| 规则 | 定义 |
|------|------|
| 最小长度 | **1 个字符**（不准空） |
| 最大长度 | 8 个中文字符 / 16 个英文字符 |
| 字符集 | 中文 + 英文字母 + 数字 |
| 禁止 | **纯数字、纯空格**、特殊符号 |
| 敏感词 | **V0 关闭**（灰盒阶段不引入平台敏感词依赖；V1 开启） |
| 唯一性 | V0 不校验重名（KK_UID 唯一）；V1 加 |

错误提示：

| 错误 | 提示文案 |
|------|---------|
| 空名 | 角色名不能为空 |
| 纯空格 | 角色名不能为纯空格 |
| 纯数字 | 角色名不能为纯数字 |
| 超长(中>8) | 角色名最多 8 个中文字符 |
| 超长(英>16) | 角色名最多 16 个英文字符 |
| 特殊符号 | 角色名不能包含特殊符号 |

校验正则（客户端+服务端双重）：
```
ALLOWED  = /^[\u4e00-\u9fa5a-zA-Z0-9]+$/   // 中文+英文+数字
拒绝：全空格、全数字、含 ALLOWED 之外字符、长度越界
```

## 五、建档（服务器）

写入初始存档（数据来自 P14 init_char_config）：KK_UID(平台)、角色ID(系统生成)、角色名(已校验)、灵根ID(1-5)、初始属性/物品/坐标/货币(P14)、version。

### 建档伪代码（节选）
```
function CreateCharacter(req):
    if not IsValidKKUID(req.kk_uid): return Error(1001,"账号信息异常")
    existing = DB.FindCharacter(req.kk_uid)
    if existing: return Success(existing, is_existing=true)   // 幂等
    if not ValidateName(req.name): return Error(1002, GetNameError(req.name))
    // V0 敏感词关，不走 SensitiveFilter
    if req.linggen_id not in [1..5]: return Error(1004,"灵根选择无效")
    char_id = GenerateCharID(req.kk_uid)
    char_data = InitFromConfig("init_char_config", config_id=1) + {name, linggen_id, version:"v0"}
    local_ok = SaveLocal(char_data); cloud_ok = SaveCloud(char_data)
    if not local_ok: return Error(2001,"本地存档保存失败")
    if not cloud_ok: ScheduleCloudRetry(char_data)   // 云端失败不阻塞，异步补写
    return Success(char_data)
```

### 错误码表
| 码 | 场景 | 客户端提示 |
|:--:|------|-----------|
| 1001 | KK_UID 无效 | 账号信息异常，请重新登录 |
| 1002 | 角色名非法 | 具体原因（空/超长/纯数字/特殊字符） |
| 1004 | 灵根无效 | 灵根选择无效，请重新选择 |
| 2001 | 本地存档失败 | 存档保存失败，请检查磁盘空间后重试 |
| 2002 | 云端存档失败 | 云端同步失败，请检查网络后重试（本地已存可先进游戏） |
| 9999 | 服务器内部错误 | 服务器繁忙，请稍后重试 |

### 建档时序（预估）
客户端→服务器 <50ms → 校验 <10ms → 初始化 <20ms → 本地存档 <100ms → 云端 <500ms → 返回 <50ms，总计 **<800ms**。云端超时(>3s)可先返回成功，异步补写。

## 六、边界条件

| 场景 | 处理 |
|------|------|
| 创角中途退出 | 不保存，下次重走创角 |
| 建档失败 | 提示"创建失败，请重试"，回创角界面 |
| 同账号双开创建 | 平台层处理，后进入者按重进流程 |
| 角色名重名 | V0 允许（KK_UID 区分），V1 加唯一性 |
