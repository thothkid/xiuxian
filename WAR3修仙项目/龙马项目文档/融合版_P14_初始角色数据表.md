# V0 初始角色数据表（融合版）

> 版本：vMerge-1.1 ｜ 2026-06-09 ｜ 对应交付清单 P14 ｜ 对应程序 S05
> 融合来源：小马《V0_初始角色数据表》(全字段/CSV·JSON/一致性核对主体) + 小龙 init_character(对比)。
> 定位：建角后服务器建档时写入的初始数据模板。表名 `init_char_config`（G3）。**初始灵石 = 100（锁定）。**

---

## ⚠ 修正说明（内部矛盾修复）

小马原稿把灵石同时写进「货币 spirit_stone=100」和「背包初始物品 1001×100」——但灵石是 CURRENCY 类，应只进货币栏、不占背包格。本融合版修正为：**灵石 100 仅作货币（spirit_stone=100），不再出现在 init_items**。背包初始物品只保留丹药。

---

## 一、初始数据（V0 单行 config_id=1）

| 字段 | 值 | 说明 |
|------|-----|------|
| `config_id` | 1 | 配置行 ID（V0 仅 1 行） |
| `hp` | 100 | 初始生命 |
| `mp` | 50 | 初始灵力 |
| `atk` | 10 | 初始攻击（单值） |
| `def` | 5 | 初始防御（单值） |
| `spirit_stone` | **100** | 初始灵石（货币，不占背包格）|
| `init_items` | `[{2002,5},{2003,3}]` | 初始背包物品：聚气丹×5 + 灵泉水×3（**已移除灵石**） |
| `spawn_x` | 0.0 | 出生 X |
| `spawn_y` | 0.0 | 出生 Y |
| `facing` | 0.0 | 出生朝向（0=正东） |
| `realm` | 练气一层 | 初始境界（固定） |
| `realm_sub` | 初期 | 初始小阶（固定） |
| `cult` | 0 | 初始修为（固定 0） |
| `linggen_id` | 0 | 默认灵根 ID（建角后由 1-5 覆盖） |
| `linggen_name` | "" | 建角后覆盖 |
| `bag_size` | 12 | 初始背包格数 |
| `version` | v0 | 数据版本号 |

> 出生点 (0,0) 落在区域表出生区中心（P18）。bag_size=12 为初始背包格数。背包 UI 默认显示 6 格（P11），但初始数据给满 12 格——初始 8 件物品（聚气丹×5+灵泉水×3）占 2 格，留 10 格给拾取测试。程序读取此值初始化背包容量，P11 的显示格数可独立配置。

## 二、配置表字段定义（init_char_config）

| 字段名 | 类型 | 必填 | 默认 | 说明 |
|--------|------|:--:|------|------|
| config_id | int | ✅ | 1 | 配置行 ID |
| hp/mp/atk/def | int | ✅ | 100/50/10/5 | 初始四属性 |
| spirit_stone | int | ✅ | 100 | 初始灵石（货币） |
| init_items | string(JSON) | ✅ | 见下 | 初始背包物品数组 `[{item_id,count}]` |
| spawn_x/spawn_y/facing | float | ✅ | 0.0 | 出生坐标与朝向 |
| realm/realm_sub | string | ✅ | 练气一层/初期 | 境界（固定） |
| cult | int | ✅ | 0 | 修为（固定 0） |
| linggen_id | int | ✅ | 0 | 默认灵根（建角覆盖） |
| linggen_name | string | ✅ | "" | 建角覆盖 |
| bag_size | int | ✅ | 12 | 背包格数 |
| version | string | ✅ | v0 | 版本号 |

### CSV 示例
```csv
config_id,hp,mp,atk,def,spirit_stone,init_items,spawn_x,spawn_y,facing,realm,realm_sub,cult,linggen_id,linggen_name,bag_size,version
1,100,50,10,5,100,"[{""item_id"":2002,""count"":5},{""item_id"":2003,""count"":3}]",0.0,0.0,0.0,练气一层,初期,0,0,,12,v0
```

### JSON 示例
```json

  "init_char_config": {
    "config_id": 1, "hp": 100, "mp": 50, "atk": 10, "def": 5,
    "spirit_stone": 100,

    "spawn_x": 0.0, "spawn_y": 0.0, "facing": 0.0,
    "realm": "练气一层", "realm_sub": "初期", "cult": 0,
    "linggen_id": 0, "linggen_name": "", "bag_size": 12, "version": "v0"
  }

```

## 三、程序读取说明

```
1. 启动加载 init_char_config → 读 config_id=1 行
2. 建角确认 → 拿初始值写入角色数据
3. 灵根 ID 在建角流程由 0 覆盖为 1-5（P13）
4. init_items 为 JSON 字符串，解析后逐项 AddItem(item_id, count)
5. spirit_stone 写入货币栏，不进背包
6. 存档字段定义见 P16
```

## 五、融合说明

| 维度 | 取自/裁决 | 理由 |
|------|------|------|
| 全字段/CSV·JSON/一致性核对 | 小马 | 完整度高 |
| **spirit_stone = 100** | 锁定裁决 | 一进游戏货币非空，便于验证货币显示 |
| **init_items 移除灵石** | 内部矛盾修复 | 灵石是 CURRENCY，不占背包格，不应同时计入 init_items |
| 初始丹药 聚气丹×5+灵泉水×3 | 小马 | 验证消耗品使用 |
| 表名 init_char_config | G3 | — |

> 否决：小龙 init_currency=0（与锁定 100 不符）、小龙 init_items=[{3,3}]（编号体系不符 G2）。
> 【thoth 待决】初始丹药是否补 1 个回灵丹。现状：聚气丹×5+灵泉水×3（小马版）。小龙闭环文档含回灵丹×1。差异影响不大（都是消耗品测试覆盖），等 thoth 拍板。
